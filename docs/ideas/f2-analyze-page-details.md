# 配色分析（F2）ページの詳細

## セクション1.カラーコード入力とPCCS近似色の編集

### 色入力UIコンポーネントの構成

- `input[type="color"]`とHEXコード用の`input[type="text"]`を縦並びにする
- `input[type="color"]`の横幅は、`input[type="text"]`の横幅と一致させる
- 初期入力値はランダムに生成

### 複数の色入力UIの配置

- 最大6色分の色入力UIを横並びに配置
- 最初は2色分表示

- 「＋」ボタンを最後の色入力UIの右に表示し、クリックしたら新たな色入力UIが追加される
- 色入力UIが6色分表示されているときは、「＋」ボタンは表示しない

- 各色入力UIに「×」削除ボタンを設置する
- 最小色数は2色とし、色入力UIが2色分のみ表示されているときは「×」ボタンを表示しない

### PCCSデータソースフィルタ

- F1と同様に「新配色カード199にある色」と「すべての色」のどちらを近似に使うか切り替えられるフィルタを設置する
- フィルタを切り替えると、すべての色のPCCS近似色・代替候補がリアルタイムで更新される

### PCCS近似色プレビュー

- 各色入力UIの下に、その色をPCCSで近似した色スウォッチを表示する
- `app/src/lib/data/pccs_colors.json`を基にCIEDE2000で最近傍PCCS値を1つ決定したものを近似色として採用
- 各色スウォッチの上に、そのPCCS表記（例：`v2`）を表示する
- ユーザーが入力した色が変わったら、対応するPCCS近似色スウォッチも更新される

### PCCS近似色の代替候補

- 各PCCS近似色スウォッチの下に、一回り小さい色スウォッチ（代替候補）を2つ横並びで表示する
- 代替候補は2件固定とし、CIEDEで2番目に近いPCCS色・3番目に近いPCCS色をそれぞれ採用する
- 各代替候補色スウォッチの下に、そのPCCS表記（例：`v2`）を表示する
- ユーザーが入力した色が変わったら、PCCS近似色・代替候補スウォッチをいずれも初期状態（1〜3番目の順）にリセットして更新する

#### 代替候補クリック時の挙動

- 代替候補スウォッチをクリックすると、クリックした色がPCCS近似色プレビューに昇格する
- 元のPCCS近似色は代替候補プールに戻り、代替候補2件は「入力色との近さ（ΔE₀₀昇順）」で左から並べ直す
  - 例：元の近似色が1番目・元の代替B（3番目）が残る場合 → 1番目が左、3番目が右に並ぶ

## セクション2.配色プレビューと色相環・トーン概念図の表示

- セクション2は、セクション1の下に表示される

### レイアウト構成

```
入力色：[入力色プレビュー  ]
PCCS近似色：[PCCS近似色プレビュー]

[色相環（SVG）] [トーン概念図（SVG）]
```

### 配色のプレビュー

- 2種類の配色プレビューを縦に並べて表示する
- 各配色プレビューの左にラベルを表示する
- 各色スウォッチにPCCS表記・HEXコードは表示しない
- 1つ目は「入力色」ラベルを付け、ユーザーが入力した色の色スウォッチを並べた配色プレビュー
  - ユーザーの入力値が変わったら、この配色プレビューも更新される
- 2つ目は「PCCS近似色」ラベルを付け、PCCS近似色の色スウォッチを並べた配色プレビュー
  - ユーザー入力値が変わってPCCS近似色が更新されたり、代替候補から別のPCCS近似色が選択された場合、この配色プレビューも更新される

### 色相環

- 配色プレビューの下に、PCCS色相環をSVGで表示する
- トーン概念図と横並びで配置する
- 2種類のハイライトを持つ：
  - **近似色ハイライト**（通常表示）
    - PCCS近似色のうち有彩色の色相セクターに枠線を追加し、彩度を高めた色で塗り直す
    - 複数色の場合は複数セクターを同時にハイライトする
    - 無彩色のPCCS近似色については何もせず、有彩色の色相だけハイライトする
    - 色相環の円の中心から、円周上のPCCS近似色の色相を結ぶ線を描画する
  - **技法範囲ハイライト**（セクション3カードホバー/クリック時）
    - その技法に関係する色相範囲のセクターを別のスタイルで強調する
    - このとき近似色ハイライトは非表示にせず不透明度を下げて薄く残し、技法範囲ハイライトを重ねて表示する
- インタラクションは持たない（read-only）。色相の変更はセクション4で行う

### トーン概念図

- 色相環の右に、PCCSトーン概念図をSVGで表示する
- トーン概念図の左から1列目は無彩色トーンで、上からW/ltGy/mGy/dkGy/Bkというラベルが書かれた正方形を並べる
- 左から2列目には上からp/ltg/g/dkg、3列目にはlt/sf/d/dk、4列目にはb/s/dp、5列目はvというラベルが書かれた円を並べる
- 各列の円や正方形は、縦中央に寄せて配置する
- 2種類のハイライトを持つ：
  - **近似色ハイライト**（通常表示）：PCCS近似色で使われているトーンのセルに枠線を追加し、彩度を高めた色で塗り直す
  - **技法範囲ハイライト**（セクション3カードホバー/クリック時）：その技法に関係するトーンのセルを別のスタイルで強調する。このとき近似色ハイライトは非表示にせず不透明度を下げて薄く残し、技法範囲ハイライトを重ねて表示する
- インタラクションは持たない（read-only）。トーンの変更はセクション4で行う

#### sトーンの表示

- sトーン（`s`）は「新配色カード199にある色」フィルタ選択時には収録外のため、セルを低彩度・低不透明度で描画し、斜線パターンを重ねて非収録であることを視覚的に示す
- 「すべての色」フィルタ選択時は通常表示する

## セクション3.配色分析結果

- セクション3は、セクション2の下に表示される

### 配色の特徴カード

- 配色を分析し、当てはまる特徴や配色技法をそれぞれカードUIとしてグリッド状に並べる
- カード内には、タイトル・解説・カテゴリタグを記載する
- PCCS近似色が変わったら、表示するカードも更新する
- 当てはまるカードが1件もない場合は「該当なし」文言を表示する
- カードの表示順は以下の見出し順に従う：
  1. 色相の関係カード（2色配色のみ表示）
  2. トーンの関係カード（2色配色のみ表示）
  3. ハーモニーカード（2色配色のみ表示）
  4. 配色技法カード
  5. 色相環の分割カード

#### カードと色相環・トーン概念図の連動

- カードをホバーまたはクリックしたとき、セクション2の色相環・トーン概念図上で「その技法に関係する色相・トーン範囲」をハイライト表示する
  - 例：「ドミナントカラー」ホバー時 → 色相差0〜3の範囲にあたる色相セクターを強調
  - 例：「トーナル」ホバー時 → `ltg`・`g`・`sf`・`d` のセルを強調
  - 例：「トライアド」ホバー時 → 等間隔3分割の色相セクターを強調
- このハイライトは通常のPCCS近似色ハイライトとは視覚的に区別する（色・スタイルを変える）
- カードからホバーが外れたとき、またはクリック解除時は通常のPCCS近似色ハイライト表示に戻る

#### 色相の関係カード

- 2色配色の場合のみ表示する
- タイトルは「同一色相」「隣接色相」…etc.
- 解説では色相差の範囲と配色の効果を説明する
- カテゴリタグとして「色相の関係」タグを表示する

#### トーンの関係カード

- 2色配色の場合のみ表示する
- タイトルは「同一トーン」「類似トーン」…etc.
- 解説ではトーンの位置関係と明度差・彩度差、配色の効果を説明する
- カテゴリタグとして「トーンの関係」タグを表示する

#### ハーモニーカード

- 2色配色の場合のみ表示する
- タイトルは「ナチュラルハーモニー」or「コンプレックスハーモニー」
- 解説では自然から学ぶ色彩調和についてや、条件と配色の効果を説明する
- カテゴリタグとして「色相の自然連鎖」タグを表示する

#### 配色技法カード

- タイトルは「ドミナントカラー」「トーナル」…etc.
- 解説では色相・トーンの条件や明度差・彩度差、配色の効果を説明する
- カテゴリタグとして「配色技法」タグを表示する

#### 色相環の分割カード

- タイトルは「ダイアード」「トライアド」…etc.
- 解説では条件と配色の効果を説明する
- カテゴリタグとして「色相環の分割」タグを表示する

## セクション4. 配色の調整

- セクション4は、セクション3の下に表示される

### 状態管理の設計

各入力色は、以下の3段階で「表示に使うPCCS色」を決定する。

```
approximatedPCCS（CIEDE2000 1番目・不変）
  ↓ セクション1での代替候補選択
selectedPCCS（セクション1の選択結果）
  ↓ セクション4の自動配色 → 個別調整 の順に適用
displayedPCCS（分析・プレビューに使われる最終的なPCCS色）
```

- 入力色（HEX）が変わると：`approximatedPCCS`・`alternatePCCS` を再計算し、`selectedPCCS` をリセット、個別調整をクリアする
- 代替候補を選択すると：`selectedPCCS` を更新し、個別調整をクリアする
- 自動配色の調整は全色の `displayedPCCS` に一括適用する
- 個別調整は対象の色の `displayedPCCS` にのみ適用する
- 調整結果の `displayedPCCS` がデータセットに存在しない場合（hue×toneの組み合わせが存在しない）、その操作をUIでdisabledにする

### 配色の調整機能

- 自動配色による全体の調整と、各色の個別調整という2つのアプローチを提供する

#### 自動配色調整

- 配色で使われているすべての色をまとめて動かす（色相移調やトーン移調を行う）ことで、配色の印象を調整する
- 以下の操作を提供する：
  - 「色相を時計回りにnずらす」または「反時計回りにnずらす」
    - Angle SliderのようなUIで実装
    - PCCS近似色で使われている色相範囲（最小包含弧）を扇形として塗りつぶし、その部分を回転操作することで、全体の色相をずらす
    - 配色プレビュー等の表示の再計算や更新はリアルタイムに行うが、履歴スタックの更新はUIからフォーカスが外れたタイミングで行う
    - セクション1でPCCS近似色が変化した場合、UIの状態をリセットする
  - 「明るくする」「穏やかにする」「暗くする」
    - いずれもボタンとして実装し、状態は持たない
    - クリックしたら配色プレビュー等の再計算や更新、履歴スタックの更新を行う

#### 個別調整

- 調整したい色を選択させ、その色の色相やトーンを動かす
- 以下の操作を提供する：
  - 「明度を上げる/下げる」「彩度を上げる/下げる」
    - ボタンとして実装し、状態は持たない
    - 操作の結果がデータセットに存在しない場合、ボタンをdisabledにする
    - クリックしたら配色プレビュー等の再計算や更新、履歴スタックの更新を行う
  - 「色相を変更する」
    - セクション4内に小さい色相環SVGを配置し、セクターをクリックして目標色相を選択する
    - 現在の色相に対応するセクターをハイライト表示する
    - データセットに存在しない hue×tone の組み合わせとなるセクターはdisabledにする
    - 色相を選択したら、配色プレビュー等の再計算や更新、履歴スタックの更新を行う
    - セクション1でPCCS近似色が変化した場合、色相環セクターの状態をリセットする

## サイドバー：配色の履歴

- 全セクションの左に、開閉可能なサイドバーを設置する

### 履歴スタック

- 以下の操作を行うごとに、その時点の `inputHex` の配列・`displayedPCCS` の配列・変更サマリー文字列を履歴スタックに積む：
  - セクション1でのinputHex変更（入力欄からfocusが外れたら履歴を更新）
  - セクション1での代替候補によるPCCS近似色の変更（すぐに履歴を更新）
  - セクション4での自動配色・個別調整の変更（Angle Slider UIはfocusが外れたら、それ以外の離散的な操作はすぐに履歴を更新）
- 履歴の記録は最大20件とし、20件を超えた場合は古いものから削除していく

### 一覧表示と復元

- サイドバーでは、履歴スタックのアイテムを新しい順に一覧表示する
- 変更サマリー文とPCCS近似色の色スウォッチを並べたプレビューからなるカード風のUIを、縦に全件分並べて表示する
- カードをクリックしたら、クリックしたカードの配色（`inputHex`・`displayedPCCS`）を復元し、各セクション1~3に反映する
  - 復元アクション自体を新しい履歴エントリとして積む（サマリーは「配色を復元」等）
  - 復元時、セクション4の調整UIの状態はリセットする
