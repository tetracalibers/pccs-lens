# F3: イメージ別配色シミュレータの構想

## 背景

色彩検定2級では、「エレガント」「カジュアル」「クラシック」…などのイメージ別配色法が試験範囲に含まれているが、PCCSを使った具体的な配色例が豊富に掲載されているテキストは少ない。

各イメージの配色ルールに沿うようにユーザーを導きながら、そのイメージに合うさまざまな配色をシミュレーションし、幾何パターンとして保存できる機能を搭載したい。

## 機能概要

イメージごとにページを用意し、次のような機能を提供する。

1. ページの冒頭には、そのイメージに合う配色をどう作るかといったルールの解説文を載せる
2. ベースカラー・アソートカラー・アクセントカラーそれぞれの色相・トーンを候補の中から選んでもらう
3. すべて選択されたら（※アクセントカラーは任意）、色の面積比率に従った幾何パターン画像（SVG）を数種類生成し、プレビューする
4. 配色に使ったPCCS色のHEXコードをコピーしたり、気に入った幾何パターン画像を保存したりすることができる

## 機能詳細

### ページ構成

#### `src/routes`配下に追加するファイル

- `+page.svelte`自体は全配色パターンで共通とし、パラメータ`theme`を使ったルートによってページを出し分ける

```
└── patterns
    ├── +page.svelte // 配色パターン一覧ページ
    └── [theme]
        └── +page.svelte // 各配色パターンのページ
```

#### 配色パターン一覧ページ`/patterns`の構成

- 既存のナビゲーションバーとトップページ（`app/src/routes/+page.svelte`）に `/patterns` へのリンクを追加する
- `app/src/routes/+page.svelte`と同様のUIで、各配色パターンページへのリンクを並べる
- 各リンク内には次の情報を含める
  - 左カラム：ベースカラーとアソートカラーを交互に並べた2x2市松模様を表示（色はベースカラー → アソートカラーの順に、サジェスト条件に沿ってランダム選択）
  - 右カラム：パターン名とそのイメージを縦に並べて表示

例）「エレガント」ページへのリンク：
```tsx
<a href={resolve("/patterns/elegant")}>
  <div class="grid-2x2">
    <span class="base-color"></span>
    <span class="assort-color"></span>
    <span class="base-color"></span>
    <span class="assort-color"></span>
  </div>
  <div class="vertical-stack">
    <span class="pattern-name">エレガント</span>
    <span class="pattern-mood">「女性的な」「気品のある」「洗練された」「優雅な」</span>
  </div>
</a>
```

#### 各配色パターンのページ`/patterns/[theme]`の構成

```tsx
<h1>カジュアルな配色</h1>
<p>（使う色の条件や配色の特徴・印象などの解説）</p>

<h2>配色プレビュー</h2>
<>（色スウォッチを並べた配色プレビューを表示）</>

<h2>色を選ぶ</h2>
<h3>ベースカラー</h3>
<p>（そのイメージではどんな色をベースカラーに選べばいいかを解説）</p>
<div style="横並び">
  <HueWheel />
  <ToneDiagram />
</div>
<h3>アソートカラー</h3>
<>（ベースカラーと同様の配置構成）</>
<h3>アクセントカラー</h3>
<>（ベースカラーと同様の配置構成）</>

<h2>幾何パターン</h2>
<div>
  <svg>
    <!-- バウハウス・パターン -->
  </svg>
  <div class="horizotal-stack">
    <button>画像を再生成</button><!-- バウハウス用 -->
    <button>PNG保存</button>
  </div>
</div>
<div>
  <svg>
    <!-- ジオメトリック・パターン -->
  </svg>
  <div class="horizotal-stack">
    <button>画像を再生成</button><!-- ジオメトリック用 -->
    <button>PNG保存</button>
  </div>

```

### 配色プレビュー

- 面積比率に従って、長方形の色スウォッチを並べる
  - 面積比率については`docs/image-based-color-rules.md`の`1-6. 面積比率`を参照
- 各長方形の上に色相記号（例：v2）を表示する（表記フォーマットは`docs/color-analysis-rules.md §2`の`[トーン記号][色相番号]`形式に準拠する）
- 各長方形の下にHEXコードとコピーボタンを並べて表示する

### 色相・トーン選択UI

#### 色の入力UIの構成

- 色相選択UIとトーン選択UIをセットで並べたものを1色用の入力UIとする
- ベースカラー・アソートカラー・アクセントカラーの3色の入力に対応する
  - 詳しくは`docs/image-based-color-rules.md`の`1-8. アクセントカラーUIの共通仕様`を参照

#### サジェスト方式

- サジェストの基本原則については`docs/image-based-color-rules.md`の`1-7. サジェストの共通仕様`を参照
- 色のデータソースは`app/src/lib/data/pccs_colors_full.json`とする
- ベースカラーが変更されたら、アソートカラー → アクセントカラーの順にサジェストを更新する
  - ベースカラーに対する相対的な条件（類似色相・アソートに彩度対比が生まれる色など）はリアルタイムで計算して反映する
  - サジェストが変わっても、現在の選択値はリセットしない
- アソートカラーが変更されたら、アクセントカラーのサジェストを更新する
  - アソートカラーに対する相対的な条件（アソートに対して色相対比や彩度対比が生まれる色など）もリアルタイムで計算して反映する
  - サジェストが変わっても、現在の選択値はリセットしない
- 配色パターンごとのサジェスト条件については`docs/image-based-color-rules.md`の`2. テーマ別定義`を参照

#### 色相選択UI

`app/src/lib/components/HueWheel.svelte`と同様のPCCS色相環を用意する。コンポーネントは `src/lib/components/patterns/HueWheelInput.svelte` として配置する。

既存の`HueWheel`と異なる点：
- 奇数番号の色相も偶数番号と同様に表示する
- 円周沿いに並ぶ色相記号の外側に、全色相の円形色スウォッチを配置し、クリックで色相を選択する
  - ただし、サジェストされる色相以外の色スウォッチは目立たない見た目とする
- サジェストされる色相の扇形は塗りつぶし、それ以外の扇形は区切り線のみ表示する（ただしクリックは可能）
- 色相を選択したら、その円形色スウォッチの枠線が太くなり、扇形にも枠線を描画し、それ以外の色相は不透明度が下がる
- 色相が選択されていても、他の色相をクリックして選択し直すことが可能
- トーン選択UIで選択したトーンが変わると、円内の扇形塗りつぶしの色や円周外の円形色スウォッチの色も、そのトーンの色に変化する
- トーン選択UIで無彩色トーンが選択された場合の挙動については、`docs/image-based-color-rules.md`の`1-9. 無彩色選択時のHueWheelの挙動`を参照

#### トーン選択UI

`app/src/lib/components/ToneDiagram.svelte`と同様のトーン概念図を用意する。コンポーネントは `src/lib/components/patterns/ToneDiagramInput.svelte` として配置する。

既存の`ToneDiagram`と異なる点：
- サジェストされるトーンのセルは塗りつぶし、それ以外のセルは視覚的に目立たない状態で表示する（ただしクリックは可能）
- トーンのセルをクリックすると、そのトーンが選択状態になる
- トーンを選択したら、そのセルの枠線が太くなり、それ以外のセルは不透明度が下がる
- トーンが選択されていても、他のトーンをクリックして選択し直すことが可能
- 色相選択UIで選択した色相が変わると、セルの塗りつぶしの色もその色相の色に変化する
- 無彩色のグレイ系トーン（ltGy,mGy,dkGy）をhoverすると、セルの左側にツールチップが現れ、細かいグレイを選択できる
  - ltGyをhoverすると、`Gy-2.5`、`Gy-3.5`が選択できる
  - mGyをhoverすると、`Gy-4.5`,`Gy-5.5`、`Gy-6.5`が選択できる
  - dkGyをhoverすると、`Gy-7.5`,`Gy-8.5`が選択できる
  - これらの細かいグレイを選択したら、選択中トーンはそのグレイのトーンとなり、セルの塗りつぶしの色もそのグレイの色に変化する

### 幾何パターン画像の生成と保存

- 正方形の幾何パターン画像をSVGで2種類生成する
  - 1つめはバウハウス・パターン：`proto/geo-pattern-generator/src/bauhaus.ts`を参考に実装
  - 2つめはジオメトリック・パターン：`proto/geo-pattern-generator/src/geometric.ts`を参考に実装
  - `generateBauhaus`や`generateGeometric`関数の引数には、次のように配色情報を渡す
    - `[ベースカラー, アソートカラー, アクセントカラー]`の配列
    - アクセントカラーが入力されていない場合は、`[ベースカラー, ベースカラー, アソートカラー]`
- 初期表示は初期値として設定された色相・トーンをもとに生成したパターン群とする（初期値の決定フローは`docs/image-based-color-rules.md §4-3`を参照）
- パターン生成処理は非同期処理として実装する。`@svgdotjs/svg.js`はDOM依存のためSSRフェーズでは動作しないので、生成処理は`onMount`または`browser`ガード内に限定する
- 「画像を再生成」ボタンはバウハウスとジオメトリックにそれぞれ個別に設置する
- 「画像を再生成」ボタンがクリックされたら、そのパターンの再生成を開始し、生成中はそのパターン画像部分にローディングを表示する
  - 他のUIは操作可能な状態を保つ
  - 生成中は該当ボタンをdisabledにして二重クリックを防止する。生成完了後にdisabledを解除する
  - 生成中に色相・トーンが変更された場合は、生成完了後に最新の色で色を置き換える

#### 色変更時の各UIの更新挙動

色相・トーンが変更されたとき、各UIは次のように更新する。

- 配色プレビュー（色スウォッチ列）：リアルタイムで更新する
- 幾何パターン：模様（図形配置・乱数シード）を保持したまま、色のみ置き換える（再生成は行わない）

#### 幾何パターン画像の保存

- 各パターン画像の下に保存ボタンを設置し、クリックしたらPNG形式でダウンロードできる
- SVGからPNGへの変換は `canvas.drawImage` + `canvas.toBlob` + `a[download]` 方式で実装する
- ダウンロードするPNGファイルの命名規則は `[theme]-[pattern-type].png` とする（例：`elegant-bauhaus.png`、`casual-geometric.png`）

テーマを10件以上に拡張した場合の一覧ページのUX（ページネーション、カテゴリフィルタなど）は現段階では考慮しない。
