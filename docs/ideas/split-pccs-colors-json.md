# @app/src/lib/data 配下のJSONファイルの整理

## 影響範囲

- @app/src/lib/data/pccs_colors.json
- @app/src/lib/data/pccs_colors_full.json

および、これらのファイルのデータを使用している @app/src 配下のコード全て

## 解決したい問題

現状、`pccs_colors.json`と`pccs_colors_full.json`は、それぞれ次のような内容になっている。

- `pccs_colors.json`：vトーン24色、vトーンとsトーン以外のトーンの偶数色相12色
- `pccs_colors_full.json`：全トーンの全色相の色

これら2つのファイルには重複して登録されている色が多く、今後の変更が容易な構成になっていない。

## 解決案

次のような構成に変更する。

- vトーン24色を登録したファイル：`pccs_colors_v_24.json`
- sトーンの偶数色相12色を登録したファイル：`pccs_colors_s_12.json`
- vトーンとsトーン以外のトーンの偶数色相を登録したファイル：`pccs_colors_12.json`
- vトーン以外のトーンの奇数色相を登録したファイル：`pccs_colors_odd_12.json`

このような構成には、次のような背景がある。

- 色彩検定では、vトーンは24色相、その他のトーンは偶数番号の12色相が出題され、sトーンは出題されない
- 当アプリでは、色彩検定の出題範囲をデフォルトとする場面が多いが、将来的に「sトーンを含める」「奇数色相を含める」オプションを追加する構想がある

このような背景から、新構成には次のようなメリットがある。

- `pccs_colors_v_24.json`：vトーンの色はPCCS色相環に使われるため、vトーンだけを分けておくことで、色相環を描画する際に無駄なデータを読み込む必要がなくなる
- `pccs_colors_12.json`：色彩検定の出題範囲内の色を対象とする処理では、`pccs_colors_v_24.json`とこのファイルだけを読み込めばよい
- `pccs_colors_s_12.json`：「sトーンを含める」オプション選択時には、このファイルだけを追加で読み込めばよい
- `pccs_colors_odd_12.json`：「奇数色相を含める」オプション選択時には、このファイルだけを追加で読み込めばよい
