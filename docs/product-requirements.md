# プロダクト要求定義書

## プロダクトビジョンと目的

> 「世界の色をPCCSというレンズを通して見る」

PCCS（Practical Color Co-ordinate System：日本色研配色体系）は、日本色彩研究所が開発した、色彩調和を目的としたカラーシステム。色相（24色相）と、明度・彩度を融合した「トーン（12区分）」の2つの要素で色を体系化し、イメージに合わせた配色や色選びを容易にする。

本アプリは、あらゆる色をPCCSの枠組みに落とし込み、色相・トーンの関係を理解・活用するためのツールを提供する。

## ターゲットユーザー

- 色彩検定（AFT）の学習者・既習者
- PCCSを用いた配色を実務で活用したいデザイナー・クリエイター

### ユーザーの課題とニーズ

| 課題 | ニーズ |
|---|---|
| 任意の色がPCCSのどの値に近いか判断できない | 色コードを入力すれば近いPCCS値がすぐわかる |
| 自分の配色がどんな配色技法に当てはまるか分からない | 複数色を入力すれば配色技法が自動で判定される |
| PCCS色相環・トーン概念図を配色検討に活かしたい | 色相環・トーン概念図上でインタラクティブに配色を調整できる |
| イメージ別の配色ルールを学びながらPCCSを使った具体的な配色を試したい | テーマ（エレガント・カジュアル等）を選んで配色ルールの解説を読みながら色を選べる。生成した配色は幾何パターン画像として保存できる |

## 主要機能一覧

| ID | 機能名 | 概要 |
|---|---|---|
| F1 | 色のPCCS近似 | 入力した色に最も近いPCCS値を上位N件表示する |
| F2 | 配色の分析と調整 | 入力した複数色をPCCSに近似し、色相環・トーン概念図上で分析・インタラクティブ調整を行う |
| F3 | イメージ別配色シミュレータ | テーマ（エレガント・カジュアルなど）ごとの配色ルールに沿って色を選び、幾何パターン画像を生成・保存する |

## 機能要件

### F1：色のPCCS近似

1. ユーザーはカラーピッカー（HEX入力兼用）で色を入力できる
2. 入力色に近いPCCS値を上位6件表示する
  - 近さの計算にはCIEDE2000（知覚的色差）を使用する
  - 各結果には色スウォッチとPCCS表記（例：`v2`）を表示する
3. 検索対象のPCCSデータソースをフィルタリングできる
  - 「新配色カード199にある色」：`app/src/lib/data/pccs_colors.json`
  - 「すべての色」：`app/src/lib/data/pccs_colors_full.json`
4. 入力色に近いJIS慣用色名を上位6件表示する
  - 近さの計算にはCIEDE2000（知覚的色差）を使用する
  - 各結果には色スウォッチ・慣用色名・読み（ふりがなまたは英語名）・出題級を表示する
5. 入力色を URL クエリパラメータ（`?color=RRGGBB`）に反映する
  - ブックマーク保存・URL 共有によって同じ色の近似結果を再現できる

### F2：配色の分析と調整

#### セクション1：カラーコード入力とPCCS近似色の編集

1. ユーザーは最大6色のカラーコードをカラーピッカー（`input[type="color"]` ＋ HEX入力欄）で入力できる
   - 最小2色・最大6色。初期表示は2色
   - 各色は「＋」ボタンで追加、「×」ボタンで削除できる（最小色数2色のとき「×」は非表示）
   - カラーピッカーの初期値はランダムに生成する
2. 検索対象のPCCSデータソースをフィルタリングできる（F1と共通）
   - 「新配色カード199にある色」または「すべての色」
   - フィルタを切り替えると、すべての色のPCCS近似色・代替候補がリアルタイムで更新される
3. 入力された各色について、選択中のデータソースを基にCIEDE2000で上位3件のPCCS近傍色を決定し、1番目を近似色・2〜3番目を代替候補として表示する
   - 代替候補スウォッチをクリックするとその色が近似色に昇格し、元の近似色は代替候補プールに戻る（ΔE₀₀昇順で左から並べ直す）
   - 入力色（HEX）が変わると近似色・代替候補をリセットして再計算する

#### セクション2：配色プレビューと色相環・トーン概念図の表示

4. 2種類の配色プレビューを表示する
   - 「入力色」プレビュー：ユーザー入力のHEXコードを並べた色スウォッチ列
   - 「PCCS近似色」プレビュー：現在の `displayedPCCS` の色スウォッチ列
5. PCCS色相環（SVG）を表示する
   - 有彩色の近似色ハイライト（セクター枠線・高彩度塗り・中心からの方向線）
   - セクション3カードホバー/クリック時の技法範囲ハイライト（近似色ハイライトは不透明度を下げて残す）
   - インタラクションは持たない（read-only）
6. PCCSトーン概念図（SVG）を表示する
   - 左端列に無彩色（W/ltGy/mGy/dkGy/Bk）、右に有彩色トーンを列構成で配置する
   - 近似色ハイライト／技法範囲ハイライト（色相環と同様）
   - sトーンは「新配色カード199」フィルタ時に非収録として視覚的に区別する
   - インタラクションは持たない（read-only）

#### セクション3：配色分析結果

7. 当てはまる配色特徴・技法をカードグリッドで表示する（詳細は「色分析ルール定義書」参照）
   - 色相の関係（2色入力時のみ）
   - トーンの関係（2色入力時のみ）
   - ハーモニー：ナチュラル/コンプレックス（2色入力時のみ）
   - 配色技法（色相・トーン軸）：ドミナントカラー、ドミナントトーン、トーンオントーン、トーナル、カマイユ、フォカマイユ、ビコロール、トリコロール
   - 色相環の分割：ダイアード、トライアド、スプリットコンプリメンタリー、テトラード、ペンタード、ヘクサード
   - 複数の技法に当てはまる場合はすべて列挙する
8. カードをホバー/クリックすると色相環・トーン概念図上に技法範囲ハイライトを表示する

#### セクション4：配色の調整

9. 自動配色による全体調整と個別調整の2種類を提供する
   - **自動配色調整**：全色まとめて色相移調（Angle Slider UI）・「明るくする/穏やかにする/暗くする」ボタン
   - **個別調整**：「明度を上げる/下げる」「彩度を上げる/下げる」ボタン、および小型色相環SVGによる色相変更
   - データセットに存在しない hue×tone の組み合わせとなる操作はdisabledにする
10. 各入力色の表示用PCCS値は `approximatedPCCS`（不変）→ `selectedPCCS`（代替候補選択結果）→ `displayedPCCS`（調整後の最終値）の3段階で管理する

#### サイドバー：配色の履歴

11. すべてのセクションの左に開閉可能な履歴サイドバーを表示する
    - 以下の操作のたびに `inputHex` 配列・`displayedPCCS` 配列・変更サマリー文字列を履歴スタックに積む（最大20件）：
      - セクション1でのinputHex変更（focus離脱時）
      - セクション1での代替候補によるPCCS近似色の変更（即時）
      - セクション4での自動配色・個別調整（Angle SliderはfocusOff時、それ以外は即時）
    - 履歴は新しい順に一覧表示し、カードをクリックするとその時点の配色を復元する（復元自体も履歴に積む）

### F3：イメージ別配色シミュレータ

1. ユーザーは `/patterns` でテーマ一覧を閲覧し、任意のテーマページ（`/patterns/[theme]`）に遷移できる
2. テーマページの冒頭には、そのテーマの配色ルール解説文を表示する
3. ベースカラー・アソートカラー・アクセントカラーそれぞれの色相・トーンをインタラクティブに選択できる
  - 色相選択UIと トーン選択UIをセットで提供し、推奨される色相・トーンをサジェストとして視覚的に示す
  - アクセントカラーは任意（初期は非表示・ボタンで追加できる）
4. 選択した色に基づき、面積比率に応じた配色プレビュー（色スウォッチ列）をリアルタイムで更新する
  - 各スウォッチにはPCCS表記（例：`v2`）とHEXコードのコピーボタンを表示する
  - 色相・トーンが変更されたとき、配色プレビューはリアルタイムで更新する
5. 選択した配色で、バウハウス・パターンとジオメトリック・パターンの2種類の幾何パターン画像（SVG）を生成・プレビューする
  - 色相・トーンが変更されたとき、幾何パターンは模様を保持したまま色のみ置き換える（再生成は行わない）
6. 各幾何パターンは「画像を再生成」ボタンで新しいランダムパターンに再生成できる
  - 再生成中は対象パターンにローディングを表示し、ボタンはdisabledにする
  - 再生成中に色が変更された場合は、生成完了後に最新の色で色を置き換える
7. 各幾何パターン画像をPNG形式でダウンロードできる
  - ファイル名は `[theme]-[pattern-type].png`（例：`elegant-bauhaus.png`）

## 非機能要件

- **パフォーマンス**：色距離計算はクライアントサイドで実行し、レスポンスタイムを最小化する
- **セキュリティ**：ユーザー入力のバリデーションを実施する（不正なカラーコード入力への対応）
- **アクセシビリティ**：色のみで情報を伝えない（ラベル・数値を併用する）
