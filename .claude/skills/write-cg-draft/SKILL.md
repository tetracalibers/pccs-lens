# CG記事草稿作成スキル

`create-cg-page` スキルで作成した CG ページの雛形（`## TODO` だけの `+page.svx`）に、**本文の草稿を執筆する**ためのスキルです。

フローは2段階です。

1. **インタビュー**: `cg-interviewer` エージェントが対象トピックを調査して**段階的なヒアリング項目**を設計し、それをもとに**メインの対話ループ（あなた）がユーザーへ実際に質問**して情報を集める。
2. **執筆**: `cg-ghostwriter` エージェントが、集めた情報をもとに `.claude/skills/write-cg-draft/style-guide.md` に従って本文を `+page.svx` に書き込む。

## 入力

- **slug**（必須）: 執筆対象ページのスラッグ（＝ディレクトリ名）。
  - 例: `/write-cg-draft camera-capture-and-cg`
  - 引数が無ければ、ユーザーにスラッグ（または対象ページのタイトル）を尋ねる。

## なぜインタビューをメインループで行うのか

`cg-interviewer` はサブエージェントであり、**ユーザーと直接対話できません**。そのため `cg-interviewer` は「何を・どの順で聞くか」という**インタビュー計画（質問リスト）**を返すだけにし、**実際のヒアリング（質問の提示と回答収集）はこのスキルを実行するあなた自身がメインの対話ループで行います**。

## 手順

### ステップ1: 対象ファイルを特定する

1. 引数の slug を使って `app/src/routes/cg/**/<slug>/+page.svx` を探す（例: `find app/src/routes/cg -type d -name "<slug>"` などで `<slug>/+page.svx` を見つける）。
2. 結果に応じて分岐する。
   - **1件見つかった** → その絶対パスを `target_file` とする。
   - **0件** → ユーザーに「該当する CG ページが見つからない」と伝える。雛形が未作成の可能性があるため、先に `create-cg-page` スキルで雛形を作るよう案内して中止する。
   - **複数件** → どれが対象か曖昧なため、候補パスを提示してユーザーに確認する。
3. `target_file` を読み、フロントマターの `title` / `group` / `draft` を取得する。
   - 既に本文（`## TODO` 以外の中身）が書かれている場合は、その旨をユーザーに伝え、**上書きしてよいか確認**してから進む。

### ステップ2: インタビュー計画を作る（`cg-interviewer`）

`cg-interviewer` サブエージェントを起動する。

- subagent_type: `cg-interviewer`
- description: `Plan interview for CG article draft`
- prompt: 次を明記する（独立コンテキストのため漏れなく渡す）。
  - `target_file`: ステップ1のパス
  - `slug` / `title` / `group`: ステップ1で取得した値
  - `seed`: ユーザーが既に伝えている方向性やメモ（あれば）

返ってきた**インタビュー計画**（トピック要約・判明済みの前提・推奨アウトライン・段階的な質問・最低ライン）を受け取る。

### ステップ3: ユーザーへ段階的にインタビューする（メインループで実施）

`cg-interviewer` の計画に沿って、**あなた自身がユーザーに質問**して回答を集める。

- **推奨アウトラインをまず提示**し、記事の方向性・節立てについてユーザーの合意を取る。
- 質問は**ステージ順**（A→B→C…）に進める。一度に大量に聞かず、段階的に進める。
- 選択肢のある質問は `AskUserQuestion` ツールを使うと答えやすい。自由記述が必要なものは通常のメッセージで尋ねる。
- 各質問には、`cg-interviewer` が用意した**仮説（たたき台）を添えて**提示し、ユーザーが「それでよい」と確認するだけでも進められるようにする。
- 回答が曖昧・不足している場合はその場で掘り下げる。必要なら回答を踏まえて `cg-interviewer` を再度起動し、追加の質問を設計してもらってもよい。
- 計画の「執筆を始めてよい最低ライン」を満たしたら、収集した内容を **`interview_summary` として整理**する（合意したアウトライン、各項目の回答、定義する用語、具体例、補足、図解・リンクの希望を含める）。
- 執筆に入る前に、`interview_summary` の要点をユーザーに提示して**最終確認**を取る。

### ステップ4: 本文を執筆する（`cg-ghostwriter`）

ユーザーの合意後、`cg-ghostwriter` サブエージェントを起動する。

- subagent_type: `cg-ghostwriter`
- description: `Write CG article draft into +page.svx`
- prompt: 次を明記する。
  - `target_file`: ステップ1のパス
  - `title` / `group`: ステップ1の値（検証用）
  - `interview_summary`: ステップ3で整理した内容**すべて**
  - `style_guide_path`: `.claude/skills/write-cg-draft/style-guide.md`

返ってきた要約（書いた節、強調した用語、プレースホルダ、要確認項目）を受け取る。

### ステップ5: 検証して報告する

1. `app` ディレクトリで `npm run check` を実行する。
2. エラー・警告が出たら、その内容を `revision_notes` として `cg-ghostwriter` を再起動し、修正させる。クリーンになるまで繰り返す（多くは未定義コンポーネントの参照や数式・ディレクティブの記法ミス）。
3. ユーザーへ次を報告する。
   - 草稿を書き込んだファイルパスと、節立ての概要。
   - `cg-ghostwriter` が挙げた**要確認項目**（推測・保留した点）。
   - `draft: true` のまま残してあること（草稿のため）。レビュー後、問題なければ `draft: true` を外して公開状態にする旨を案内する。

## 注意事項

- このスキルは**既存の `+page.svx`（雛形）に本文を書く**ためのもの。新規ページ（ディレクトリと雛形）の作成は `create-cg-page` の担当。
- フロントマター（`layout` / `title` / `group`）は変更しない。`draft: true` は人間のレビューが済むまで残す。
- サブエージェント（`cg-interviewer` / `cg-ghostwriter`）は独立コンテキストで動作するため、起動ごとに必要な情報をすべて prompt に含める。
- 実際のユーザーへの質問は**必ずメインループ（あなた）が行う**。サブエージェントにユーザー対話をさせない。
- 執筆スタイルは `.claude/skills/write-cg-draft/style-guide.md` を唯一の基準とする。
- 作業後は `app` ディレクトリで `npm run check` を実行し、エラー・警告が出ないことを確認する。
