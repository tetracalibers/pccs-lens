# 要求内容: F4 慣用色名比較セクション

## 背景・目的

`/jis-color-map/[family]` のページでは既に `JisColorMap` によって色み全体の慣用色名マップが表示されているが、試験対策として「似た色を見分ける練習」を支援する導線はまだ無い。

そこで同ページのマップの下に、`jis_colors.json` 各 subfamily の `compareSections` に基づいた**慣用色比較セクション**を並べる。各比較セクションはミニマップで比較対象を強調しつつ、慣用色プレビュー・情報・色相/明度/彩度の比較図・解説を組み合わせ、似た色同士の違いを視覚化する。

## 対象データ

`jis_colors.json` の各 `JISSubfamilyData` に既に `compareSections: JISCompareSection[]` が存在する。型定義は `app/src/lib/data/jis-colors.ts` の次のとおり。

```ts
export type JISCompareSection = {
  targets: string[]        // 比較対象の JISColor.id（1件以上）
  hintJIS?: string[]       // 参考として示す JISColor.id（任意）
  hintPCCSHue?: number[]   // 参考として示す PCCS 色相番号（任意）
}

export type JISSubfamilyData = {
  colors: JISColor[]
  compareSections: JISCompareSection[]
}
```

## 変更・追加する機能

### 変更ページ

- **`/jis-color-map/[family]`** の既存 `JisColorMap` の下に、subfamily 単位の比較セクション群を追加する
  - subfamily ごとに `JISSubfamily.name` を見出しにしたブロックを作り、その subfamily の `compareSections` 順に比較セクションを並べる
  - 対象 subfamily は `getSubfamiliesByGroup(family.id)` で取得した順番を尊重する

### 新規コンポーネント

以下を `app/src/lib/components/jis-color-map/` 配下に新規作成する。

1. **慣用色比較セクション** — 1 件の `JISCompareSection` を表示するセクション（ミニマップ＋慣用色一覧＋比較図＋解説）
2. **慣用色名ミニマップ** — 比較対象を強調表示したコンパクト版マップ
3. **コンパクト慣用色スウォッチ** — ミニマップ内で `targets`・`hintJIS` 含まない慣用色を表示する正方形セル（ラベルはホバー時にツールチップで表示）
4. **参考慣用色スウォッチ** — ミニマップ内で `hintJIS` を表示する正六角形セル
5. **色み比較図 / 明度比較図 / 彩度比較図** — マンセル色相・明度・彩度が異なる場合にそれぞれ表示される比較矢印図

### ロジックファイル

コンポーネント固有でない、比較処理に関するロジックは `app/src/lib/jis-color-map/` 配下に TS ファイルとして切り出す。マンセル値関連のロジックは `app/src/lib/color/munsell.ts`、データ取得は `app/src/lib/data/jis-colors.ts` に追加する。

## ユーザーストーリー

- **色彩検定の学習者として**、似た色の違い（色み・明度・彩度のどの軸で異なるのか）を視覚的に理解したい。慣用色名マップを眺めるだけでは気付きにくい微妙な違いを意識的に比較したい。
- **色彩検定の学習者として**、ある慣用色がどの色相域に属するかを俯瞰したい。比較セクション内のミニマップで比較対象を強調表示することで、マップ全体の中での位置関係も同時に把握したい。

## 慣用色比較セクションの画面構成

```
┌──────────────────────────────────────────────────────┐
│ [慣用色名ミニマップ]   │ [慣用色一覧]                │
│                      │  - 級タグ + プレビュー + 情報 │
│                      │  - 色み/明度/彩度の比較図   │
│                      │  - 慣用色解説                │
└──────────────────────────────────────────────────────┘
```

- デスクトップでは左にミニマップ、右に一覧
- モバイルではミニマップの下に一覧（縦並び）

### 慣用色名ミニマップの要件

- subfamily に属する慣用色を色相・明度順に並べたコンパクト版マップ
- セルサイズは通常の `JisColorMap` より小さく
- `targets` に含まれる慣用色は**コンパクト慣用色スウォッチ**で目立つように表示
- `hintJIS` に含まれる慣用色は**参考慣用色スウォッチ（正六角形）**で表示
- `targets` にも `hintJIS` にも含まれない慣用色は opacity 0.2 で薄く表示
- `hintPCCSHue` に含まれる PCCS 色相は通常表示、それ以外の PCCS 色相セルは opacity 0.2 で薄く表示

### 慣用色一覧の要件

- `targets` に含まれる慣用色を順に表示
- 各慣用色について以下を横並びで表示
  - 級タグ（`examLevel` を表す小さなバッジ）
  - プレビュー（角丸正方形を HEX で塗りつぶし）
  - 情報（`name`・`systematicName`・`munsell` を中央寄せ縦並び、文字サイズは name > systematicName > munsell）
- `targets` 内でマンセル色相・明度・彩度が異なる場合、該当する比較図（色み/明度/彩度比較図）を並べて表示
- 各 `target` の `colorDescription` を解説として表示（空白で改行）

### 色み比較図（マンセル色相が異なる場合のみ表示）

- 上下両端にマンセル色相寄りのラベル（「赤」「黄」「緑」「青」「紫」）を配置
- 両端の間を両方向矢印で埋める（コンテナ高さいっぱいに広がる）
- 矢印は両端ラベルに対応する色のグラデーション

### 明度比較図（マンセル明度が異なる場合のみ表示）

- 上下両端に「低明度」「高明度」ラベルを配置（相対的判断）
- 間に片方向矢印（高明度方向に矢をつける）
- 矢印は高明度方向に向かって明るくなるグレイのグラデーション
- `targets` が 3 要素以上で明度順に並んでいない場合、中間にも明度ラベルを挟み「ラベル - 矢印 - ラベル - 矢印 - ラベル」という 3 段構成にする

### 彩度比較図（マンセル彩度が異なる場合のみ表示）

- 上下両端に「低彩度」「高彩度」ラベルを配置（相対的判断）
- 間に片方向矢印（高彩度方向に矢をつける）
- 矢印は高彩度方向に向かって鮮やかになる `ColorFamily` 色相のグラデーション
- `targets` が 3 要素以上で彩度順に並んでいない場合、中間にも彩度ラベルを挟み 3 段構成にする

## 受け入れ条件

1. `/jis-color-map/red` など各 family ページで、既存の `JisColorMap` の下に subfamily ごとの比較セクション群が表示される
2. subfamily の見出しラベルが `JISSubfamily.name` になっている
3. 各 `compareSections` エントリにつき 1 つの慣用色比較セクションが描画される
4. 比較セクションのミニマップで、`targets` がコンパクト慣用色スウォッチ、`hintJIS` が参考慣用色スウォッチ（正六角形）で強調表示される
5. `targets`・`hintJIS` に含まれない慣用色セル、`hintPCCSHue` に含まれない PCCS 色相セルが opacity 0.2 で薄く表示される
6. 慣用色一覧に `targets` 各色の級タグ・プレビュー・情報・解説が表示される
7. `targets` 内でマンセル色相/明度/彩度が異なる場合のみ、それぞれの比較図が表示される
8. `targets` 要素数 3 以上で明度/彩度が順序どおりに並んでいない場合、該当比較図に中間ラベルが追加される
9. モバイル幅（640px 以下程度）ではミニマップと慣用色一覧が縦並びになる
10. ミニマップ内のコンパクト／参考慣用色スウォッチは、初期状態では色名を表示せず、hover またはキーボードフォーカス時のみ独自ポップオーバーで色名を表示する（色を見て名前を予想 → 答え合わせの学習 UX）
    - `nameSegments` が指定されている場合は要素ごとに改行して表示
    - 参考慣用色スウォッチ（正六角形）は、色の付いた六角形領域のみがホバー反応する（矩形の外側をホバーしてもポップオーバーは開かない）
11. `colorDescription` に空白を含む場合、空白位置で改行されて表示される
12. `npm run check`（型チェック）・`npm run lint`・`npm run format` がすべてパスする

## 制約事項

- コンポーネント固有でないロジック（比較処理・順序判定・ミニマップ構築の共通部分など）は `app/src/lib/jis-color-map/` 配下に TS ファイルとして切り出す
  - 既存 `JisColorMap` と新規ミニマップで重複する処理も TS に寄せる
- マンセル値関連のロジック（色相/明度/彩度の比較・ラベル生成など）は `app/src/lib/color/munsell.ts` に実装する
- データ取得ロジック（id から JISColor を引く、等）は `app/src/lib/data/jis-colors.ts` に実装する
- `app/src/lib/color/` と `app/src/lib/data/` 配下の既存関数を積極的に利用する
- 新規 Svelte コンポーネント（このページ専用）は `app/src/lib/components/jis-color-map/` 配下に配置する
- 命名はアイデアメモの疑似コードを参考にしつつ、既存コードベースの命名規約に合わせて適切なものを検討する
- アイデアメモ中のコード例はすべて疑似コードであり、変数名・Props 名は仮称
- 既存コンポーネント（`JisColorMap.svelte`・`JisColorSwatch.svelte`・`PccsSwatch.svelte`・`ValueSwatch.svelte`）と `buildJisColorMap` は原則として変更しないが、ミニマップとの重複処理を切り出す過程で内部のユーティリティ参照先を共通 TS に差し替える程度の編集は許容する（挙動は変えない）
