# 要求内容: F4 慣用色名一覧リストページの実装

## 概要

慣用色名マップ（`/jis-color-map`）に新規ページ「慣用色名一覧リストページ（`/jis-color-map/all`）」を追加する。全慣用色名をマンセル色相順（起点2RP）で一覧表示し、由来・PCCS近似値・系統色名・マンセル値などの詳細情報を提供するとともに、既存のトップページに一覧ページへのリンクカードを追加する。

---

## 追加内容

### 1. 新規ページ：慣用色名一覧リストページ（`/jis-color-map/all`）

- パンくずリストとページ見出しの下に、全慣用色のアイコンをマンセル色相順（起点2RP）で並べて表示する
  - アイコンは各慣用色の `iconKey` から `JIS_COLOR_ICON_MAP.get(jisColor.iconKey)` で取得したアイコン識別子を用い、その慣用色の `hex` で塗りつぶして表示する
  - 各アイコンは、ページ内の該当慣用色名詳細セクション（`id` 属性付き見出し）へのアンカーリンクとして機能する
- アイコン群の下に、慣用色名詳細セクションをマンセル色相順（起点2RP）で並べる
  - 無彩色はマンセル色相の定義外であるため、有彩色の後ろに配置する
- 並び順は次のルールに従う（有彩色）:
  1. `JISColor.munsell` の色相部を `munsellHueRank` でランク化して昇順ソート（起点2RP）
  2. 色相が同じ場合は明度が高い順
  3. 明度も同じ場合は彩度が高い順
- 無彩色（`JISColor.munsell` が `N` で始まるもの）は有彩色の末尾に配置し、明度が高い順に並べる

### 2. 新規コンポーネント：慣用色名詳細セクション

各色系統（`ColorFamily`）単位で慣用色名の詳細情報をまとめて表示するコンポーネント。

Props:
- `family: JISColorFamily`
- `jisColors: JISColor[]`（そのfamilyに属し、並び順が確定した慣用色の配列）

各慣用色ごとに表示する情報（1色 = 1エントリ）:

- アイコン（`JIS_COLOR_ICON_MAP.get(jisColor.iconKey)`）
- 角丸正方形の色スウォッチ（`jisColor.hex` で塗りつぶし）
- 慣用色名（`jisColor.name`。要素に `id={jisColor.id}` 属性を付与。ページ上部アイコン群からのアンカーリンクターゲットとなる）
- 読み（`jisColor.reading`）
- PCCS近似値（`jisColor.approximatePccs` を列挙。各記法を既存の `PccsSwatch` 相当で表示）
- 由来説明（`jisColor.originDescription`。空白文字を含む場合はその位置で改行して表示）
- 系統色名（`jisColor.systematicName`）
- マンセル値（`jisColor.munsell`）
- 比較ページへのリンク（`/jis-color-map/{family.id}/` に遷移。「{family.name}を比較する」等のラベルとリンクアイコン）
- 級タグ（`jisColor.examLevel` が `2` または `3` の場合のみ表示。`null` のときは非表示）

レイアウト:
- デスクトップでは「アイコン・色スウォッチ・名前」エリアと「由来・詳細情報」エリアを横並びにする
- モバイルでは縦並びにする

### 3. 新規コンポーネント：級タグ（共通化）

`JisColorCompareSection.svelte` 内に書かれている級タグ（`examLevel` を `2級` / `3級` と表示するバッジ）を独立した共通コンポーネントとして切り出す。

- Props: `examLevel: 2 | 3 | null`
- `null` のときはスペーサーを出力する（既存 `JisColorCompareSection.svelte` と同じ挙動）
- スタイルは既存の `.level` / `.level-2` / `.level-3` / `.level-none` と同等にする

### 4. 新規コンポーネント：慣用色名一覧リストページへのリンクカード

既存の `FamilyCard.svelte` と同様の外観・構造を持つカードコンポーネント。

- リンク先は `/jis-color-map/all`
- `.card-checker` の色模様は、各 `ColorFamily`（7系統）からそれぞれランダムに1色ずつ選んだ計7色を使う
  - `FamilyCard.svelte` の `.card-checker` は2色4セルのグリッドだが、このカードは7色に対応したグリッド構成とする（グリッドの列数・行数は設計フェーズで確定）
- タイトルは「慣用色名 全一覧」、説明文は「由来やPCCS近似値など、色の詳細を学ぼう」とする

### 5. 変更：慣用色名マップトップページ（`/jis-color-map`）

- 既存の `FamilyCard` グリッドの上に「一覧で覚える」セクションを追加する

```
<h2>一覧で覚える</h2>
<p>由来やPCCS近似値など、色の詳細を学ぼう</p>
<慣用色名一覧リストページへのリンクカード />

<h2>比較して覚える</h2>
<p>似た色を比較して覚えよう</p>
<div class="grid">
  {#each familyCards as card (card.family.id)}
    <FamilyCard {...card} />
  {/each}
</div>
```

---

## 背景・理由

既存の慣用色名マップ（F4）は、family 別比較ページ（`/jis-color-map/[family]`）が中心であり、似た色の比較に特化している。一方、色彩検定の学習者は「すべての慣用色名を俯瞰しながら、由来・PCCS近似値・マンセル値などの個別情報を同時に参照したい」というニーズを持つ。この一覧ページを設けることで、比較学習（既存）と網羅的な詳細学習の両方をカバーできるようになる。

---

## ユーザーストーリー

- **色彩検定の学習者として**、全慣用色名をまとめて一覧で確認したい。それぞれの色の由来やPCCS近似値、マンセル値を一か所で確認でき、記憶の定着につなげたい。
- **色彩検定の学習者として**、上部に並んだアイコンをクリックして、見たい色の詳細セクションに素早くジャンプしたい。
- **色彩検定の学習者として**、慣用色名一覧を眺めながら、似た色の比較ページにすぐ移動できるリンクを利用したい。

---

## 受け入れ条件

1. `/jis-color-map/all` にアクセスするとページが表示される
2. ページの `<title>` が「慣用色名 全一覧 — PCCS Lens」になっている（実装時に既存ページのタイトル命名パターンとの整合を確認する）
3. ページ上部にパンくずリスト（慣用色名マップ > 全一覧）が表示される
4. 全慣用色のアイコンが、マンセル色相順（起点2RP）で並んで表示される
5. 各アイコンはその慣用色の HEX 色で塗りつぶされている
6. 各アイコンをクリック（またはキーボードで選択）すると、そのページ内の詳細セクションの慣用色名見出しにスクロール遷移する
7. アイコン群の下に慣用色名詳細セクションがマンセル色相順で並んでいる
8. 各慣用色名のエントリに「アイコン」「色スウォッチ」「慣用色名（`id` 属性付き）」「読み」「PCCS近似値」「由来説明」「系統色名」「マンセル値」「比較ページへのリンク」が表示される
9. `originDescription` に空白が含まれる場合、空白の位置で改行されて表示される
10. `examLevel` が `2` または `3` の慣用色には級タグ（バッジ）が表示される。`null` の場合は非表示（スペーサーとして扱う）
11. 慣用色名詳細セクションの「比較ページへのリンク」が `family` ページ（`/jis-color-map/[family.id]/`）に正しく遷移する
12. `/jis-color-map` のトップページに「一覧で覚える」セクションと慣用色名一覧リストページへのリンクカードが追加されている
13. リンクカードのタイトルが「慣用色名 全一覧」、説明文が「由来やPCCS近似値など、色の詳細を学ぼう」で表示される
14. リンクカードの `.card-checker` が全7系統からランダムに1色ずつ選んだ7色の模様で表示される
15. デスクトップでは詳細セクション内のレイアウトが横並びになり、モバイル幅では縦並びになる
16. `npm run check`（型チェック）・`npm run lint`・`npm run format` がすべてパスする

---

## 制約事項

- コンポーネント固有でないロジック・ユーティリティは `app/src/lib/jis-color-map/` 配下に TS ファイルとして切り出す
  - マンセル値の色相ランク変換（`munsellHueRank`）は既存の `app/src/lib/color/munsell.ts` を使用する
  - データ取得ロジックは `app/src/lib/data/jis-colors.ts` に実装する
- `app/src/lib/color/` と `app/src/lib/data/` 配下の既存関数を積極的に利用する
- 新規 Svelte コンポーネントは `app/src/lib/components/jis-color-map/` 配下に配置する
- アイデアメモ中のコード例はすべて疑似コードであり、変数名・Props名は適切に命名する
- 無彩色（`JISColor.munsell` が `N` で始まるもの）はマンセル色相が定義されないため、有彩色の末尾に配置し、明度が高い順に並べる
- 既存コンポーネント（`FamilyCard.svelte`・`JisColorCompareSection.svelte`）の外部インタフェース（Props型・スロット）は変更しない
- 級タグの共通コンポーネント化にあたり、既存 `JisColorCompareSection.svelte` 内のインラインスタイルを共通コンポーネントへの参照に置き換えてよい（挙動・見た目は変えない）
