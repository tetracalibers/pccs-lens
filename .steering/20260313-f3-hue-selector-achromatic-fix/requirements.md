# 要求内容：無彩色選択時のHueSelectorサジェスト・スウォッチ表示の修正

## 背景

`feature/f3-palette-suggest` ブランチにて、テーマに基づいた色相・トーンのサジェスト機能を実装中。
HueSelectorにおいて、無彩色トーンが選択されたときの表示に以下の問題があった。

## 問題点

1. **サジェスト表示の過剰表示**
   無彩色トーン（W / ltGy / mGy / dkGy / Bk）が選択されると、HueSelectorの全24色相が
   サジェスト扱い（不透明度1.0）で表示されていた。
   本来は `allowedHues`（テーマで使用可能な色相）のみをサジェスト表示すべき。

2. **円形色スウォッチの色が単色になる**
   無彩色選択時、全スウォッチが選択中の無彩色のHEX（例: Bkなら #1a1a1a）で塗られてしまい、
   各色相の識別ができなかった。

3. **スウォッチ色がトーンに連動しない**
   有彩色トーン選択時も、スウォッチが `HUE_COLORS`（固定の代表色）を使っており、
   選択トーンの色（例: pトーンならそのpトーンの各色相の色）に連動していなかった。

## 要求

- 無彩色選択時の不透明度制御を `allowedHues` ベースに変更する
- 円形色スウォッチの色を常に各色相の実色（トーン連動）で表示する
- 有彩色選択時も無彩色選択時も、スウォッチ色は `getHueColor(h)` で統一する

## 受け入れ条件

- 無彩色選択時、`allowedHues` に含まれる色相のみ不透明度 1.0 で表示される
- 無彩色選択時、`allowedHues` に含まれない色相は不透明度 0.2 で薄く表示される
- Bkトーン選択時、スウォッチが全て黒になることはない
- pトーンなど有彩色トーン選択時、スウォッチが各色相のpトーン色を反映する
