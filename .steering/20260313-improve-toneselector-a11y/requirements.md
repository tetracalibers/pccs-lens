# 要求内容：ToneSelector アクセシビリティ改善

## 変更・追加する機能の説明

`ToneSelector.svelte` のグレイ細分ツールチップに対するキーボード操作とARIA属性を整備し、アクセシビリティを向上させる。

## ユーザーストーリー

- キーボードのみで操作するユーザーが、グレイバケットセル（ltGy / mGy / dkGy）をフォーカスしてツールチップを開き、サブトーンを選択できる
- スクリーンリーダーユーザーが、セルがポップアップを持つことと、その展開状態を把握できる

## 受け入れ条件

- ツールチップが開いたとき、最初のサブトーンボタンに自動でフォーカスが移る
- `ArrowDown` / `ArrowUp` でツールチップ内のボタン間を循環移動できる
- `Escape` または `Tab` でツールチップが閉じ、フォーカスがトリガーセルに戻る
- グレイバケットセルに `aria-haspopup="menu"` と `aria-expanded` が付与されている
- ツールチップ要素に `role="menu"` が付与されている
- サブトーンボタンに `role="menuitemcheckbox"` と `aria-checked` が付与されている
- `Space` キーはセルの開閉・選択に割り当てない（ページスクロールとの競合を避けるため）

## 制約事項

- CSS Anchor Positioning を使った既存のツールチップ配置ロジックは変更しない
- SVG 内の `<g>` 要素には `anchor-name` を直接設定できないため、既存の不可視 HTML アンカー（`.cell-anchor`）の仕組みはそのまま維持する
