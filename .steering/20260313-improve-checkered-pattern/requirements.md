# requirements.md — `/patterns`ページ市松模様の色選択アルゴリズム改良

## 変更・追加する機能

`app/src/routes/patterns/+page.svelte` に表示されている各テーマの市松模様（2×2グリッド）の色選択を改良する。

現状は `computeSuggest`（一般的なサジェストルール）からランダム選択しているため、次の問題がある。

- ベースとアソートで同色が選ばれ単色になるケースがある
- テーマのイメージが伝わりにくい色の組み合わせになるケースがある

## ユーザーストーリー

- ユーザーとして、`/patterns` ページ一覧を見たとき、各テーマカードの市松模様から各テーマのイメージが直感的に伝わってほしい

## 受け入れ条件

- 各テーマに専用の「市松模様用サジェストルール」（`effectiveRules`）が定義されている
- `effectiveRules` は一般ルールより制限が強く、テーマイメージを顕著に表す色相・トーンのみを候補とする
- ベースとアソートで同色が選ばれることがない（色相または無彩色バケットが異なる）
- `/patterns` ページの市松模様はこの専用ルールを使って色を選択する
- 既存の `computeSuggest` や `THEMES` 定義には変更を加えない

## 制約事項

- `effectiveRules` はアイデアメモ（`docs/ideas/f3-improve-checkered-pattern.md`）で定義された内容を使用する
- 型は既存の `SuggestOutput` を再利用する
- `/patterns/[theme]` ページ（個別テーマページ）の挙動は変更しない
