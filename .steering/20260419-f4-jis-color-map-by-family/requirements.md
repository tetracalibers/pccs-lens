# 要求内容: F4 慣用色名比較マップ（色みごとのページ）

## 背景・目的

既に `/jis-color-map` に「全色」のマップが表示されているが、全130色を一覧するため情報密度が高く、色み（`ColorFamily`）ごとの俯瞰や比較には向いていない。

本作業では、`ColorFamily` 単位で分割された色みごとの慣用色名マップページ（`/jis-color-map/[family]`）を新規作成し、既存の「慣用色名マップ一覧ページ」（`/jis-color-map`）をその入口となるファミリーへのリンク集に変更する。

同一の `JisColorMap` コンポーネント（`groupId` で描画対象を切り替える）を再利用することで、DRY な実装を保ちながら7色みの個別ページを提供する。

## 変更・追加する機能

### 変更ページ

- **`/jis-color-map`（慣用色名マップ一覧ページ）**
  - 現在表示されている `JisColorMap`（`groupId = "red"` を表示する誤った状態）を削除
  - 代わりに、`/jis-color-map/[family]` へのリンクカードを `/patterns` ページと同じデザインで配置
  - 各カードの市松模様は、そのファミリーに属する慣用色からランダムに2色選んで塗る

### 新規ページ

- **`/jis-color-map/[family]`（色みごとの慣用色名マップページ）**
  - `[family]` には `ColorFamily` 型の値（`red` / `brown` / `yellow` / `green` / `blue` / `purple` / `neutral`）が入る
  - 対応する `family` に属する慣用色のマップを `JisColorMap` で表示
  - ページ冒頭に「〜系の慣用色名マップ」という `Heading1` 見出し
  - 見出しの下（または上部近辺）に `Breadcrumb` コンポーネントでパンくずリストを表示
    - 「慣用色名マップ > 〜系」
  - 不正な `[family]` 値が指定された場合は 404

## ユーザーストーリー

- **色彩検定の学習者として**、赤系・青系など色みごとに慣用色名をまとめて眺めたい。全色マップは情報量が多すぎて、特定の色みの覚えたい色に集中できないため。
- **色彩検定の学習者として**、一覧ページから各色みのマップに素早く遷移したい。また、現在どの色みを見ているかをパンくずで確認したい。

## 画面構成

### `/jis-color-map`（一覧ページ）

- `Heading1`: 「慣用色名マップ」（既存と同じアイコン・文言を維持）
- リンクカードのグリッド（`/patterns` と同じデザイン）
  - 各カードは `/jis-color-map/[family]` に遷移
  - カード左側: 2x2 の市松模様（ファミリー所属色からランダム選出した2色で塗り分け）
  - カード右側: ファミリー名（日本語 + 英語識別子）と短い説明文
- 一覧ページ自体からは元々あった `JisColorMap` の描画を削除する

### `/jis-color-map/[family]`（色みごとのページ）

- `Breadcrumb`: `慣用色名マップ > 〜系`
- `Heading1`: 「〜系の慣用色名マップ」
- `JisColorMap` に `groupId = family` を渡して描画

## ファミリーのラベル・説明文

- 日本語ラベルは既存の `JIS_COLOR_FAMILIES[i].name` を利用
- 英語ラベルや説明文が必要な場合は実装時に適切な内容を決定する
  - 参考: 赤系 / 茶系 / 黄系 / 緑系 / 青系 / 紫系 / 無彩色系

## 受け入れ条件

1. `/jis-color-map` にアクセスすると、7つの `ColorFamily` に対応するリンクカードがグリッド表示される
2. 各リンクカードは `/patterns` ページと同じデザイン（市松模様 + テキスト）である
3. 各カードの市松模様は、そのファミリーに属する慣用色から2色ランダムに選んで塗り分けられている
4. カードをクリックすると `/jis-color-map/[family]` に遷移する
5. `/jis-color-map/[family]`（例: `/jis-color-map/red`）にアクセスすると、対応するファミリーの慣用色名マップが表示される
6. 各ファミリーページには「〜系の慣用色名マップ」という `Heading1` 見出しとパンくずリスト（`慣用色名マップ > 〜系`）が表示される
7. パンくずリストの「慣用色名マップ」リンクをクリックすると一覧ページに戻れる
8. 不正な `[family]` 値（例: `/jis-color-map/unknown`）に対しては 404 エラーが返る
9. `/jis-color-map` ページから既存の `JisColorMap` の表示が削除されている
10. `npm run check` で型エラーが出ない
11. リント（`npm run lint`）・フォーマット（`npm run format`）が通る

## 制約事項

- コンポーネント固有でないロジック・ユーティリティは `app/src/lib/jis-color-map/` 配下に TS ファイルとして切り出す
  - マンセル値関連のロジックは `app/src/lib/color/munsell.ts`
  - データ取得関連のロジックは `app/src/lib/data/jis-colors.ts`
- 新規 Svelte コンポーネント（今回のページ専用のもの）は `app/src/lib/components/jis-color-map/` 配下に配置
- `app/src/lib/color/` 配下と `app/src/lib/data/` 配下の既存関数を積極的に利用
- 既存の `JisColorMap.svelte` / `buildJisColorMap` は変更せず、`groupId` の受け入れ範囲だけで再利用
- `ColorFamily` 型の値の妥当性検証は `+page.ts` の `load` 関数または `+page.server.ts` で行い、不正値は `error(404, ...)` を投げる
- 命名（変数名・Props名・コンポーネント名）はアイデアメモの疑似コードを参考にしつつ、既存コードベースの命名規約に合わせて適切なものを検討する
- アイデアメモ中のコード例はすべて疑似コードであり、変数名やプロパティ名は仮称として扱う
