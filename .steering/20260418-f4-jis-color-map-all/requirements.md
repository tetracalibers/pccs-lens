# 要求内容: F4 慣用色名比較マップ（全色）

## 背景・目的

F4 機能群の一部として、JIS慣用色（130色）を一覧表示し、PCCS色相との関係性を視覚的に比較できるマップビューを提供する。

現状、慣用色名は `jis_colors.json` に格納されているものの、ユーザーが「どの色相・明度にどの慣用色が位置するか」を俯瞰的に把握する手段がない。本機能では、マンセル色相を横軸・マンセル明度を縦軸として色を格子状に配置した「慣用色名マップ」を導入する。

将来的に家族（family）・サブファミリー（subfamily）単位での比較ビューも構築する前提があり、`groupId` を切り替えて描画対象を変えられる再利用可能なコンポーネントとして設計する。今回の作業では `groupId = "all"`（全色）のみを対象として実装する。

## 変更・追加する機能

### 新規ページ

- **`/jis-color-map`** — 全JIS慣用色を並べたマップを表示するページ

### 新規コンポーネント（`app/src/lib/components/jis-color-map/`）

- **慣用色名マップ** — `groupId: ColorFamily | ColorSubfamily | "all"` を受け取り、該当色を色相・明度順に並べたマップを描画する
- **明度スウォッチ** — 明度スケール/等明度軸のセルとして使用される
- **慣用色スウォッチ** — 慣用色を表すセル
- **PCCS色スウォッチ** — 参考表示するPCCS色相を表す円形セル

### 新規ロジック（`app/src/lib/jis-color-map/`）

- 描画対象色の集約、色相範囲・明度範囲の算出、セル配置の決定などの計算ロジック
- マンセル関連のユーティリティは `app/src/lib/color/munsell.ts` に追加
- データ取得関連は `app/src/lib/data/jis-colors.ts` に追加

### 変更

- **`/`（サイトトップ）** — 「コンテンツ」セクションに `/jis-color-map` へのリンクを追加

## ユーザーストーリー

- **色彩検定の学習者として**、JIS慣用色の全体像をマンセル色相・明度の2軸で俯瞰したい。どの慣用色名がどのあたりの色域に位置するかを視覚的に理解したい。
- **色彩デザイナーとして**、似た色相・明度の慣用色を並べて比較し、微妙な差異（彩度の違い、PCCS基準色との距離）を把握したい。

## マップの構造

### 軸

- **横軸**: マンセル色相（`10RP` を最も若い値としてソート）
- **縦軸**: マンセル明度（`2.5` から `9.0` まで `0.5` 刻み、上に行くほど高明度）
  - 明度 `0.5` が必ず含まれるよう、最大・最小明度を決定する

### 補助要素

- **明度スケール（左端の列）**: 明度スウォッチを縦に並べる
- **等明度軸（明度0.5の横列）**: 横軸色相の個数分の明度0.5のスウォッチを並べる
- **参考PCCS色（helpPCCSList）**: 描画対象慣用色が属する全subfamilyの `hintPCCSHue` 範囲内の vトーン色（奇数色相含む）。PCCS色スウォッチとして配置される

### セル配置規則

- 同明度に複数色ある場合は、マンセル彩度の低いものほど上に配置（同彩度は同じ縦位置）
- 明度0.5の色は等明度軸上の明度0.5スウォッチを置き換えて配置
- 無彩色は明度スケール上の明度スウォッチを置き換えて配置
- 同一マンセル値の慣用色は1つの慣用色スウォッチにまとめる（複数名を縦に並べて表示）
- 慣用色と参考PCCS色のマンセル値が一致する場合は、慣用色スウォッチ側にPCCS色記号を併記

## スウォッチの仕様

### 明度スウォッチ

- 正方形を `scale` 倍だけ縦に引き伸ばした長方形（`scale` 未指定時は1）
- 角丸・塗りなし
- 枠線と文字色は `var(--color-body)`
- 中央に `明度${X.X}` のラベル

### 慣用色スウォッチ

- 角丸正方形、慣用色のHEXで塗りつぶし
- 中央に慣用色名（`nameSegments` があれば要素ごとに改行）
- 複数色名が与えられた場合は縦に並べて表示
- PCCS記号が与えられた場合は正方形の上に表示
- 文字数に応じて文字サイズを調整し正方形内に収める
- 文字色は `isLightColor` 関数で見やすい色に分岐

### PCCS色スウォッチ

- 円形、PCCS色相の vトーンHEX で塗りつぶし
- 中央に `${色相番号}:${色相記号}` を表示
- 文字数に応じて文字サイズを調整
- 文字色は `isLightColor` 関数で見やすい色に分岐

## 受け入れ条件

1. `/jis-color-map` にアクセスすると、全130色のJIS慣用色がマンセル色相×明度の格子状に配置されたマップが表示される
2. マップの左端には明度スケールが表示され、明度0.5の横列（等明度軸）が水平に描画されている
3. 各subfamilyの `hintPCCSHue` 範囲内にあるvトーンのPCCS色が、参考色として円形スウォッチで配置されている
4. 同一マンセル値の慣用色は1つのセルにまとめて表示される
5. 同一マンセル値に慣用色と参考PCCS色が両方ある場合、慣用色スウォッチ内にPCCS記号が併記される
6. 無彩色の慣用色は明度スケール上に配置され、明度スウォッチを置き換える
7. 画面幅が狭くマップがはみ出す場合、横スクロールできる
8. サイトトップページ `/` の「コンテンツ」セクションに `/jis-color-map` へのリンクが追加されている
9. ページには `patterns/+page.svelte` と同様のデザインの見出し（`Heading1`）が配置されている
10. `npm run check` で型エラーが出ない
11. リント（`npm run lint`）・フォーマット（`npm run format`）が通る

## 制約事項

- 新規ロジックは `app/src/lib/jis-color-map/` 配下にTSファイルとして切り出す
- マンセル値関連のユーティリティは `app/src/lib/color/munsell.ts` に追加
- JIS慣用色データ取得関連は `app/src/lib/data/jis-colors.ts` に追加
- 新規Svelteコンポーネントは `app/src/lib/components/jis-color-map/` 配下に配置
- `app/src/lib/color` 配下および `app/src/lib/data` 配下の既存関数を積極的に利用する
- ページのレイアウトでは `max-width` を指定しない（マップが広いため）
- 命名（変数名・Props名・コンポーネント名）はアイデアメモの疑似コードを参考にしつつ、既存コードベースの命名規約に合わせて適切なものを検討する
