# 要求内容：フォーカスリングのアクセシビリティ改善

## 背景

SVGの `<path>`・`<circle>`・`<g>` 要素に `tabindex="0"` を付与すると、ブラウザはデフォルトの矩形フォーカスリングを描画する。HueSelectorの扇形やToneSelectorの円形・矩形セルに対してこれが適用されるため、形状と一致せず不自然に見える。

## 変更機能

キーボードフォーカス時のフォーカスインジケータを、ブラウザデフォルトの矩形から各要素の形状（扇形・円・矩形）に合わせたカスタムSVGインジケータに置き換える。

## ユーザーストーリー

- キーボードユーザーとして、Tab キーで HueSelector の扇形をフォーカスしたとき、扇形の形状に沿ったフォーカスリングが表示されることを期待する
- キーボードユーザーとして、Tab キーで ToneSelector のセルをフォーカスしたとき、セルの形状（円または矩形）に合わせたフォーカスリングが表示されることを期待する
- マウスユーザーとして、クリック操作時に不自然な矩形フォーカスリングが表示されないことを期待する

## 受け入れ条件

- [ ] HueSelector の扇形をキーボードフォーカスしたとき、扇形に沿った青破線のフォーカスリングが表示される
- [ ] HueSelector のスウォッチをキーボードフォーカスしたとき、円形の青いフォーカスリングが表示される
- [ ] ToneSelector の円形セルをキーボードフォーカスしたとき、円形の青破線フォーカスリングが表示される
- [ ] ToneSelector の矩形セル（無彩色）をキーボードフォーカスしたとき、矩形の青破線フォーカスリングが表示される
- [ ] ToneSelector のグレイ細分ツールチップのサブアイテムをフォーカスしたとき、矩形の青破線フォーカスリングが表示される
- [ ] 選択中の要素をフォーカスしても、選択インジケータが崩れない（選択インジケータが優先）
- [ ] ブラウザデフォルトの矩形フォーカスリングは非表示になっている

## 制約事項

- aria 属性・role・tabindex など既存のアクセシビリティ属性は変更しない
- マウスクリックの動作に影響を与えない
