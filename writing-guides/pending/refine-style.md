# refine-style（保留）

## このファイルの位置づけ

`refine-style.md`（推敲・修正傾向）の保留プール。根拠が単一コミット・単一記事に閉じる、修正方向が一定しない、一般的な編集技法に還元される、などの理由で、まだ主要ルールに採用していない（または除外した）修正傾向の観察を集めた補助記録である。

- ここに置かれた項目は **文章生成・推敲時には適用しない**（`author-style-writer` はこのファイルを読まない）。
- `author-style-analyzer` が分析のたびにこのファイルを読み、根拠が増えた項目を `refine-style.md` の主要ルールへ**昇格**する／新たな保留を**追記**する／棄却・除外する対象とする。

次の操作は、根拠が単一コミット・単一記事に閉じる、修正方向が一定しない、初稿執筆であって推敲差分でない、一般的な編集技法に還元される、などの理由で主要ルールにしない。生成・推敲時に積極的には適用しない。

- **ページタイトルと重複する最上位見出しの削除・小見出しの昇格**：主張の中核を明確に示すのはPCCS色相環の描き方（ba048ff → 4c0c741）の1件のみ。他記事の根拠は節タイトルの短縮や、逆に小見出しを増やす方向の編集で、「フラット化」に一貫しない。H1を本文で繰り返さない一般的慣習とも重なるため保留する。

- **用語を使う直前に短い定義を差し込む**：既存本文への定義挿入として明確なのはPCCSによる色表示（d8d6c80 → f9a39de、「色票集」の定義文追加）の1件のみ。もう一方（色名の種類）は ComingSoon→初稿執筆で、推敲時の「差し込み」ではない。用語を使う前に定義するのは普遍的な文章術でもあり、著者固有と判断できないため保留する。

- **冗長な前置き・口語的フィラーの削除／中立化**：基本色彩語（「イメージとしては」の削除）とPCCS色相環の描き方（c1155a0 → 29194fa、「覚え方は人それぞれですが」→「さまざまな覚え方がありますが」）の2件のみで、操作も削除と言い換えで異なる。一般的な推敲技法であり、著者固有と言い切れないため保留する（一人称の砕けた前置きは残しており、口語を一律に排除するわけではない）。

- **情報の密な一文の分割**：属性ごとに段落を割る例（色相環と色立体 29194fa → 81239a3）がある一方、同じPCCS色相環の描き方の別コミットでは逆に2行を1行へ結合しており、分割／結合の方向が固定できない。方向性を持つルールとして成立しないため保留する（意味のまとまり単位で都度判断していると見られる）。

- **現象・具体を示してから用語を命名・定義する順への並べ替え**：等色相面の命名を図の後ろへ移す1件（色相環と色立体 29194fa → 81239a3）のみで、単一インスタンスでは一般化できないため保留する。完成記事全体の「現象→命名」構成は `writing-style.md`「命名後置の構成」の担当。

- **末尾に適用範囲・限界の注意を補い断定を弱める**：色彩体系と表色系（165d8af → 8b9ffa3、「このようなニュアンスから」→「からか」）のヘッジ追加は1文字追加の1件のみ。もう一方（色名の種類の末尾注意）は初稿執筆で推敲差分ではない。ヘッジ語「あくまでも」等の常用は推敲で足したのか元の書き癖かを差分だけでは切り分けられない。適用範囲の注意付加とヘッジ化という別操作を束ねている点も含め保留する。断定を弱めるヘッジ表現そのものは `stylistic-quirks.md`「ヘッジ文末」「限定・留保の言い回し」で扱う。

### CG基礎シリーズで観察したが保留する修正

- **ソース行の改行整理・段落分割**：カメラでの撮影とCGでは関連文の結合（複数箇所）、CGと画像処理では定義文と対比文の分割や段落分割（空行追加）が見られるが、結合／分割の方向が一定せず整形層の操作のため保留する（意味のまとまり単位で都度判断していると見られる）。

### 視覚・色覚シリーズで観察したが保留する修正

- **抽象的な仕組みに具体例（Example/Note）を追加する**：抽象的な原理を説明する節へ「〜を見たとき：…」のような具体場面を Example として足す操作。だが distinctive な「具体的な場面の Example」は色覚説1記事（赤／橙／明るい橙、反対色説節の3例）にほぼ依存し、反対色応答の 555nm Note は裏付け・正当化で「具体的な場面」ではない。明るさではむしろ Note（100点満点テスト）が削除されており一様でない。単一記事の局所パターンのため保留し、corpus 拡大時に再検証する。
- **用語導入で「〜を〜という」と「〜は〜と呼ばれる／呼ばれます」を交互に使い反復を避ける**：眼の構造で硝子体・眼底写真・毛様体は「〜と呼ばれます」へ、瞳孔・虹彩・中心窩は「〜といいます」のまま残るが、使い分けに規則性が見えず単一記事のため保留する。
- **ASCII擬似コード（M = 共通部分 + …）を LaTeX 数式（$$…$$）へ整形し途中式を展開する**：反対色応答の単一記事のみ、かつ技術的表記の整形寄りで文体修正と切り分けが必要なため保留する。
- **例え・対応づけを「〜のような役割を果たします」で和らげる緩衝表現の追加**：眼の構造の「水晶体＝カメラのレンズ」記述のみの単発事例のため保留する。
- **修辞的な問いかけ（「〜でしょうか？」）を節の転換点に置いて次の説明へ導く操作**：明るさの単一記事のみで、stylistic-quirks 寄り（→「節頭の反語的な問いかけ」）のため保留する。

### 混色シリーズで観察したが保留する修正

- **末尾にまとめた図を該当セクションへ移し、抽象的説明を図の操作・観察の指示に書き換える**：加法混色と減法混色（13da8e3→4a050ab）で、CardGrid後にまとまっていた PrimaryColorsWheel・MixingComplementary を補色／三原色のPCCS近似セクションへ移し、ReflectCMY を参照説明の前へ移し、絵の具の波長の言葉だけの説明を「上の図の、Rの帯とGの帯の『吸収』の部分を重ねてみてください」へ書き換えた。ただし根拠は1記事のみで、「上の図の…重ねてみてください」はこの記事固有のスペクトル帯デモに強く依存する。図の移動自体は一般的な編集改善で著者固有性が薄い。読者へ観察・操作を促す命令形「〜してみましょう／みてください」はコーパス全体で頻出する著者の癖だが、それは refine の変換ではなく通常の文体特徴で `stylistic-quirks.md`「誘い掛け」が担う。refine としては根拠不足のため保留する。
- **具体例・図の観察を先に置き、その後で『このように』と一般化する順に並べ替える**：加法混色と減法混色（4a050ab）で「このように、色料でさまざまな色をつくるには…」を、赤×緑が黒に近くなる具体的なExampleの後ろへ移した。しかし1記事1箇所のみで、「このように」＋一般化はコーパス全体で確立した書き方であり、この並べ替えは既定パターンへ寄せただけとも読める（削除・Example新設に伴う副産物の可能性も排除できない）。refine の並べ替え傾向としては根拠不足で保留し、構成傾向は `writing-style.md`／`thinking-flow.md` が高確度で扱う。
- **直前の説明と重複する冗長な言い換えを削除する**：加法混色と減法混色（4a050ab）で、直前段落と内容が重複する一文を削除した1件のみ。同区間で Example の新設・「このように」の移動など大きな組み替えが同時に起きており、削除がその副産物である可能性を排除できない。冗長な言い換えの削除は普遍的な編集行為でもあり、1例では著者固有の傾向として一般化できないため保留する。既存「冗長・重複した説明や過剰な例えを削る（自己推敲）」の一例としては吸収しうる。
- **断定的な否定を和らげる書き換え**（「単純に予測することはできない」→「…簡単ではありません」、実世界 5b7219f、単一例）、**口語的な略称を正式用語へ置換**（「RGBの混色量の組」→「色光の三原色の混色量の組」、等色実験 847c8cc、単一例・技術的正確化とも解釈可）、**説明を因果・積み上げ順へ並べ替え**（カラー印刷の発色原理の再構成、実世界 5b7219f、単一セクション）、**パラグラフ結合**（等色実験 847c8cc、微細）、および混色量記号の慣習統一・数式マークアップ化・「色がつく」→「黒くなる」等の技術訂正は、いずれも単一例・技術整形・微細な整形のため文体傾向としては採用せず保留する。

### デジタル画像シリーズで観察したが保留する修正

- **推敲時に記事内の表記を漢字・カタカナへ寄せて統一する**：ベクタ表現とラスタ表現（f6a53ff→8e705e2）で「じつは」→「実は」（漢字化）、「升目」→「マス目」（カタカナ化）。単一記事の少数例で、しかも一方は漢字へ・他方はカタカナへと方向が逆で「寄せて統一」という単一方向ルールとして一貫しない。どちらへ寄せるかの表記の好みは `stylistic-quirks.md`「ひらがな書き選好」周辺の領域であり、既存「用語・表記の統一」（記事内で揃える操作）には具体語の一括ルールとして残さず保留する。

### CG・幾何変換シリーズで観察したが保留する修正

- **意味のまとまりで段落を組み替える（フレーミング文は分離、密結合の論証は連結）**：一文ごとに改行された草稿を意味のまとまり単位で再段落化し、「ここでは〜を見ていきます」等の移行・フレーミング文は空行で独立段落に切り出し、「〜ため」「つまり」「逆に」で密に連なる論証は改行を詰めて一段落にまとめる操作（変換の合成＝intro の移行文を独立段落化・論証3文を1段落へ連結、アフィン変換＝階層説明の連続文を連結、座標系 42f5a61＝球面座標のθ・φ説明の2文を連結）。ただし多くの「連結」は mdsvex のソフト改行（\n→半角空白）の詰めで、意図的な意味まとまり再段落化と差分だけからは切り分けにくく、連結と分離の両方向が観察され判断基準が一意に確定できない。媒体依存の交絡を認めた限定下でのみ有効で、著者固有の段落戦略と断定するには根拠が弱いため保留する。同種の現象はグラデーション生成・塗りつぶし処理（CG基礎の描画記事）でも観察でき、話題転換での段落分割は `2` 記事、密接な `2` 文の同一行への連結は `1` 記事（グラデーション生成）で確認できるが、.svx のソフト改行に起因する整形の交絡が同じく残るため保留は変えない。

### 照明・光シリーズで観察したが保留する修正

このシリーズ（近接した色の影響・主観色・照明と照明光・照度と照明設計・ランプの種類・光のエネルギーと明るさ）には、使役形コミットで記録されるAI初期草稿が一切存在しない（git log で確認済み）。ここでの観察はすべて著者自身の自己推敲コミット（「文体の調整」「文章微調整」等）から採ったもので、AI草稿→人手編集の差分ではない。したがって「AIらしさの除去」の根拠にはならない。なお、この系統で確認できた常体→敬体化と句点補充は本体「常体（である調）の下書きを敬体（ですます調）へ整える」の根拠として反映済み。

- **口語的・近似的な語を正確なドメイン用語へ置換する**：ランプの種類（9f984f6「文章微調整」）で、白色LEDの分光分布の説明を「R・G・Bのうち、Rの波長域…他の2色の波長域にツノがある」→「長波長域…短波長域・中波長域には突出したピークがある」、「Rの波長が増強」→「長波長域のエネルギーが増強」へ置換する。ただし根拠は1記事・1コミット・2箇所のみで、他5記事に同種例がない。口語的・近似的表現を正確な専門用語へ言い換える推敲は技術文書一般で行われる標準的作業で、記事テーマ（スペクトル分布）固有の精度要求による訂正とも説明でき、著者固有の癖と判断する根拠に乏しい。本体「語の厳密性・説明のスコープを補正する」の一例として吸収しうるが、単一記事のため主要ルールには昇格せず保留する。
- **対比を接続詞「一方」から逆接「〜が」に畳んで一文化し、通念→主要情報の順に並べ替える傾向の可能性**：照明と照明光（76f2035）で「人工光にはさまざまな種類がある／一方、自然光は太陽光のみである」→「自然光は太陽光ですが、人工光にはさまざまな種類があります」。1例のみで一般化不可、要追検証のため保留する。
- **抽象的な説明の直後に日常的な具体例を足して接地する傾向の可能性**：照明と照明光（76f2035）で「ろうそくの光も人工光です」を追記、照度と照明設計で「たとえば、光源を遠くに置くと…」を提示。ただし後者は直接の before/after 差分ではなく完成文中の記述で確度不足。本体「数値例・具体場面の追加」の近縁だが、直接差分は実質1例のみのため保留する。
