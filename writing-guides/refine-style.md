# refine-style（推敲・修正傾向）

## ガイドの目的

このガイドは、下書きを本稿へ仕上げる際の「修正傾向」を扱う。追加・削除・並べ替え・書き換え・構成変更のうち、Git差分から確認できる操作を記述する。

## 根拠の性質に関する重要な注意

本来 `refine-style.md` は「AI草稿と人間による編集との差分」から修正傾向を抽出するファイルである。ただし本ガイドの根拠には、性質の異なる2系統が混在する。この違いが各ルールの確度と適用範囲を左右するため、先に区別しておく。

- **実際のAI草稿→人手編集の差分**：`[ai-draft]` を含むコミット（`〜 [ai-draft]` / `〜：修正内容 [ai-draft]`、および後方互換の使役形「〜：草稿を書かせた」）でAI生成文が記録され、その後に人手編集が重なっている記事から採れる差分。この系統に限り、AIらしいメタ足場・自己言及・冗長な再要約・定型表現の除去を、本ファイル本来の趣旨どおり修正傾向として扱える。
- **著者自身の自己改稿（下書き→推敲）の差分**：AI草稿が存在せず、著者が段階的に手書きした下書きから完成稿への推敲を追った差分。ここから採れるのは、箇条書きメモの散文化、因果・目的の補足追加、記法・用語の統一といった著者自身の推敲操作に限る。対象となるAI草稿がないため、「AI特有の定型表現の除去」はこの系統では根拠にできない。

この2系統を踏まえ、次の制約を置く。

- **AI特有の定型表現（メタ足場・自己言及・冗長な再要約）の除去は、実際のAI草稿差分がある記事の根拠に限って扱う**。自己改稿のみが根拠の操作へは一般化しない。
- 根拠が単一シリーズ・近接時期に閉じる傾向は、確度を **条件付きの傾向以下**に抑え、「強い傾向」「一貫して」等の強い表現は使わない。
- 各根拠のコミット参照は、修正前後の対応を示す最小限の識別子として、各ルールの「根拠」欄にのみ記載する。差分の生ログは含めない。
- 根拠が単一のコミット・単一の記事に閉じる操作は、主要ルールにせず `pending/refine-style.md` へ分離する。
- AI草稿マーカーを持つコミットが存在しない記事群からは、AI草稿→人手編集の修正傾向は確認できない。その場合は無理に推定せず、根拠なしとして扱う。

なお、下書き段階から完成稿まで一貫して現れる構成（現象→命名の全体構成など）は「推敲による修正」ではないため `writing-style.md` の担当であり、本ファイルでは推敲時に新たに生じた差分だけを扱う。

---

## 修正時の基本方針

観察されるのは、下書きの説明を本稿として整えるにあたり、**曖昧さを減らし、理解の足場を足し、表記を揃える**方向の修正である。逆に、純粋な列挙や、密接に連続する一動作などは、無理に書き換えずそのまま残す。修正の方向（分割するか結合するか等）は一定せず、意味のまとまり単位で都度判断していると見られる。

## 書き換える傾向

### 箇条書き→散文（純粋な列挙は箇条書きのまま残す）

- 対象：下書き段階で説明メモとして置いた箇条書き。
- 修正前の特徴：説明内容が箇条書きメモのまま並んでいる。
- 修正後の傾向：接続表現でつないだ地の文へ書き換える。`:::Note` のような独立注記も、関連説明の直後の地の文へ溶かし込むことがある。
- 修正操作：箇条書き→散文化。ただし項目が並列で数え上げられる内容（名称の羅列、属性の対応など）は箇条書きのまま残す。
- 修正の目的：説明を流れのある本文にしつつ、一覧性が要る列挙は一覧のまま保つ。
- 差分の種類：表現の書き換え／構成の変更。
- 適用する状況：概念解説で、説明的なメモを本文へ仕上げるとき。
- 適用しない状況：項目が対等に並ぶ列挙。手順解説（手順本文はもともと散文で書かれ、この変換は観測されない）。
- 確度：条件付きの傾向（直接確認できる事実）。根拠は2記事だが、1記事内で複数節にわたり反復し、明確な境界条件（純粋な列挙は残す）を伴う。
- 根拠：
  - マンセル表色系（下書き 3aa9ba3 → 推敲 a4b2b8c）：明度・彩度・色立体・表示方法の各節が箇条書き→散文、`:::Note` を本文へ統合。
  - PCCSによる色表示（1b66b9d → d8d6c80）：等色相面の箇条書き1項目→リンク付き散文。
  - 視覚・色覚シリーズ（反対色応答 7012163→7f63600／色覚説 c95ef39→edf7738／明るさ efc1f6c→c44ad78）：説明・理由のメモ箇条書きを接続表現でつないだ地の文の段落へ展開する（反対色応答の赤緑／明るさ／黄青チャンネル各節、色覚説の三色説・反対色説・段階説各節、明るさの視物質・順応各節）。これは下書き→完成の中心的操作で、概念解説の自己推敲というスコープ内では強い傾向だが、根拠が同一著者・同一時期・色概念解説に偏るため、他タイプ・第三者草稿へは一般化しない。
  - 実世界でみられる混色（下書き d85f930 → 推敲 5b7219f）：カラーモニタ／カラー印刷／織物／絵の具の4セクションすべてで、草稿の常体・体言止めの箇条書き（「〜である」「〜わかる」等）がです・ます体の散文段落へ書き換えられている（例：「カラーモニタの発色原理は…併置加法混色である」→「…3色の併置加法混色がカラーモニタの発色原理です」）。ただし直接の変換根拠はこの1記事・1回の編集イベントに限られる（他の改稿記事は草稿段階から散文で、変換対象がない）ため、確度は上げない。
- 反例・例外：基本色彩語（114a68c → ddb995b）は進化の7段階を、色相環と色立体（76a57d6 → 29194fa）は三属性の対応を、いずれも箇条書きのまま最終稿に残す。すべての箇条書きが散文化されるわけではない。視覚・色覚シリーズでも、並列項目・用語定義（明るさの明順応／暗順応・明所視／暗所視・赤/緑/青オプシン、反対色応答冒頭の3信号リスト、色覚説の「三色説では説明がつかない現象」リスト）は箇条書きのまま残り、その多くは「名詞：説明」のコロン形式へ整えられる。ただしコロン形式化が明確なのは明るさ1記事のみで弱い傾向。混色シリーズでも境界条件は成立し、加法混色と減法混色（4a050ab）では「物体がシアンに見える ＝ …」「加法混色の補色：Rと[C]…」などの並列・対の列挙は箇条書きのまま保持される。デジタル画像シリーズでは逆方向（散文→箇条書き）も観察される：画像の量子化はビット数の積み上げ説明（「1ビットは…2ビットなら…3ビットなら…」の散文）を3項目の箇条書きへ変換し（72ef77d→f8e815c）、逆にカラー画像とデータ量はR/G/B成分の3項目箇条書きを「それぞれ R・G・B チャンネル…と呼びます」の散文へ折り込む（61a1bc2→a7f1307）。ただし各方向 n=1・同一シリーズで、カラーの list→prose は書きかけ状態を経た大幅リライトの副産物とも読めるため、双方向の変換ルールとしては弱い傾向にとどめ、「並列に数え上げられる内容か／流れで説く内容か」で都度判断する点だけを保つ（方向を固定ルール化しない）。CG・幾何変換シリーズでは地の文→箇条書きの方向が複数記事で確認でき、AI草稿が地の文で「AはX、BはY」と並べた対比・分類・複数条件・導出の各項を箇条書きへ変換し、その際に項目を「〜です／ます」の完全文ではなく体言止めや常体（「〜する」「〜だけ」「〜になる」）の簡潔な形にそろえる（座標系 42f5a61＝y軸の向き・右手/左手系、7cb9b91＝隣辺/対辺・球面/円柱の導出を語尾「求められる」の常体で、アフィン変換＝階層分類を「です」文→「長さと角度を保つ」の体言/常体へ・直線性/平行性を箇条書き、鏡映 70d68da＝y=x鏡映の係数決定手順を箇条書き）。ただし因果・条件で密結合する一続きの論証はむしろ地の文へ連結し、すべての並列を箇条書きにはしない。

### 地の文化で長くなった節を ### 小見出しで下位トピックへ分割する

- 対象：地の文へ展開されて長くなった節。
- 修正前の特徴：一つの節に複数の下位トピックが小見出しなしで連なる。
- 修正後の傾向：### 小見出しを追加して下位トピックへ分割する。小見出しは「暗順応：明るい場所から暗い場所へ」のように主題＋補足のコロン形式を用いることがある。
- 修正操作：### 小見出しの追加。
- 修正の目的：長くなった節の見通しを保つ。
- 差分の種類：構成の変更。
- 適用する状況：地の文化（→「箇条書き→散文」）で節が長くなったとき。
- 適用しない状況：元から短い節・短い記事。眼の構造は各節が短く小見出し追加なし、反対色応答は追加が限定的。
- 確度：条件付きの傾向（直接確認できる事実）。色覚説（三色説の妥当性と限界／反対色説の妥当性）と明るさ（6箇所）の2記事で確認。コロン形式小見出しは明るさ1記事のみで一般化しない。
- 根拠：色覚説（c95ef39→edf7738）、明るさに対する感度と順応（efc1f6c→c44ad78）。
- 関連ルール：地の文化の操作は本ファイル「箇条書き→散文」（本項はその副産物として節が長くなった場合の分割）。

### 並列する数式・定義を TermCard/CardGrid に整理し、必要なら復習カードを挿入する

- 対象：地の文中に並んだ、対になる／並列する複数の数式・定義。
- 修正前の特徴：正逆の変換式、各方向のスキュー・鏡映行列、2D版と3D版の行列、各変換の式が地の文中に並ぶ。
- 修正後の傾向：`::::CardGrid` ＋ `:::TermCard{title=...}` の見出し付きカードにまとめる。導出の前提として過去に扱った式が要るときは「復習：〜」という TermCard を新たに挿入することもある。
- 修正操作：並列数式のカード化、必要なら復習カードの挿入。
- 修正の目的：並列・参照用の式群を、本文の連続的な導出から切り分けて可読にする。
- 差分の種類：構成の変更。
- 適用する状況：数式を伴う実装解説で、並列関係にある複数の式があるとき。
- 適用しない状況：導出の主線となる中心的な1本の式（カード化せず地の文の $$...$$ のまま置く）。並列関係にない単独の式。
- 確度：条件付きの傾向（直接確認できる事実）。座標系 7cb9b91（各変換式を TermCard 化）、基本的な変換と同次座標 e5a7143（2D/3D 行列を TermCard 化）、鏡映とスキュー（x軸/y軸/y=x/3D鏡映・各スキューをカード化＋「復習：拡大・縮小」カードを新設）、アフィン変換 812fc52（各変換式を TermCard 化）で確認。TermCard/CardGrid は当リポジトリ固有のディレクティブ（媒体依存）で他媒体へは移植不可。根拠はCG・幾何変換の単一シリーズに閉じる。
- 反例・例外：導出の主役となる中心的な1本の式はカード化しない。
- 関連ルール：完成記事での並列公式変種のカード配置は `writing-style.md`「並列カード：同格の兄弟概念を横並びにする」（並列公式変種→横並びカード、長い段階的導出→cols=1）。記法は `syntax-guide.md`。

### 指示語・主語の明示化

- 対象：指示語だけで受けていた主語・目的語。
- 修正前の特徴：「これらは」「これらの平面には」のように、指す対象が一意に決まりにくい指示語。
- 修正後の傾向：それが何を指すかを明示する語を足す（「これらの色票集は」「断面として現れるこれらの平面には」）。文をまたぐ動作の主語に指示語を補うこともある。
- 修正操作：指示語への先行詞の補足、または主語の明示。
- 修正の目的：誤読の余地を消す。
- 差分の種類：表現の書き換え。
- 適用する状況：推敲時、指示対象・主語が一意に決まりにくい箇所。
- 適用しない状況：直前の名詞が一意で誤読の余地がない場合は補わない。
- 確度：条件付きの傾向（直接確認できる事実）。3記事で確認できるが、操作は不均一（先行詞の具体名詞化・修飾語追加・主語の新規補足）。一般的な明確化技法とも重なる。
- 根拠：
  - マンセル表色系（a4b2b8c → 4d1bab0）：「これらは」→「これらの色票集は」。
  - 色相環と色立体（29194fa → 81239a3）：「これらの平面には」→「断面として現れるこれらの平面には」。
  - PCCS色相環の描き方（c1155a0 → 29194fa）：主語を明示して文を分割。
- 反例・例外：文脈上ただ一つに定まる指示語はそのまま残す箇所も多く、すべての指示語を必ず補うわけではない。

### 曖昧・一般的な記述を文脈に応じて具体化する

- 対象：意味の曖昧な一文や、文脈に対して一般的すぎる語（AI草稿→編集差分）。
- 修正前の特徴：「画素の位置をこの `t` に対応づけて」のように操作が具体化されていない一文や、直感的な説明部で文脈に対し一般的すぎる語（「値」）が使われている。
- 修正後の傾向：曖昧な一文を、始点・終点や具体式（左から `i` 番目の割合は `i / (N - 1)`）まで示す記述へ書き換える。直感的な説明部の一般語「値」は「色」へ寄せ、必要なら「`2` 点の値（その画素の色）」と括弧で補う。
- 修正操作：曖昧な記述の具体語・具体式への置換。
- 修正の目的：読者が手順・対象を具体的に追えるようにする。
- 差分の種類：表現の書き換え／説明の粒度調整。
- 適用する状況：直感的説明の一文が曖昧なとき、または一般語が文脈で具体化できるとき。
- 適用しない状況：形式的・数学的な文脈では一般語「値」を残す（`f(t)` や輝度は「値」のまま）。すべての抽象語を機械的に具体化はしない。
- 確度：条件付きの傾向（直接確認できる事実）。ただし根拠はグラデーション生成の単一記事に閉じ、値→色の語彙置換は個別性が高く単一系統のみで弱い。曖昧記述の具体化自体は一般的な編集術とも重なり、著者固有性は限定的。
- 根拠：グラデーション生成（AI草稿 65bcd5b→0aec308→5128584）：「`2` 点の値」→「`2` 点の値（その画素の色）」の括弧補足、「画素の位置をこの `t` に対応づけて」の曖昧な一文を削って `i / (N - 1)` の具体式まで示す段落を新設。
- 反例・例外：数式・形式部では「値」を保持する使い分けがあり、方向は「直感部は具体語へ・形式部は値のまま」で一定しない。
- 関連ルール：具体例・具体場面の新設による具体化は本ファイル「数値例・具体場面の追加」。技術的な正確化を伴う語の置換は本ファイル「語の厳密性・説明のスコープを補正する」。

### 用語・表記の統一

- 対象：同じ対象を指す語の表記ゆれ・用語ゆれ。
- 修正前の特徴：記事内で同じ対象を複数の語・表記で指している。
- 修正後の傾向：記事全体で一つに揃える方向へ置換する。
- 修正操作：用語の統一（「中心軸」→「無彩色軸」）、カタカナ表記の統一（「カテゴリー」→「カテゴリ」）。
- 修正の目的：記事内の一貫性を保つ。
- 差分の種類：表現の書き換え。
- 適用する状況：推敲時、同一対象の表記・用語がゆれている箇所。
- 適用しない状況：同じ語でも指す対象が異なる場合は統一しない。機械的な全置換はしない。
- 確度：条件付きの傾向（直接確認できる事実）。複数箇所にわたる置換が確認できるが、各具体置換は単一記事ずつ。
- 根拠：
  - 色相環と色立体（29194fa → 81239a3）：彩度の説明で「中心軸」→「無彩色軸」を複数箇所で統一。
  - 基本色彩語（114a68c → ddb995b）：「色カテゴリー」→「色カテゴリ」を3箇所以上で統一。
  - CG・幾何変換シリーズ：和語を漢字表記へ寄せる正規化（「つねに」→「常に」、「さきほど」→「先ほど」、基本的な変換と同次座標 c4dc96c・c916324）と、「拡大縮小」→区切りを入れる正規化（記事により「拡大・縮小」中黒／「拡大／縮小」スラッシュ）を確認。ただし漢字化の根拠は実質1記事にとどまり、区切り記号は同一記事内・記事間で中黒とスラッシュが混在して方向が定まらない。「拡大縮小→区切り挿入」という方向はあっても、区切り記号の種類（・か／か）まで一律に決めつけない条件付き扱いとする。
  - グラデーション生成（AI草稿 65bcd5b→5128584）：口語的なかな表記や外来語を一段フォーマルへ寄せる。かな→漢字（「となりあう」→「隣り合う」、「もっとも」→「最も」、「しかた」→「仕方」）、口語→術語（「やり方」→「手法」）、外来語→和語（「アーティスト」→「作り手」）。ただし「なめらか」は残る箇所と「滑らか」へ寄る箇所が混在し、寄せは不徹底。
- 反例・例外：縦切りの説明で有効な「中心軸」は別の節で残しており、文脈で有効な語は置換しない。CG・幾何変換シリーズでは区切り記号の選択（中黒／スラッシュ）が未確定で、記号の種類までは統一ルール化しない。かな→漢字化や口語→術語の寄せは一般的な推敲・校正とも重なり著者固有性は弱く、比較的著者らしいのは外来語→和語（「アーティスト」→「作り手」）の語選択だが単一例にとどまる。
- 関連ルール：長音符の有無（カテゴリー／カテゴリ）といった表記の好みそのものは `stylistic-quirks.md`「ひらがな書き選好」周辺の表記の癖と重なる。ここでは「推敲時に記事内で揃える操作」として扱う。なお `stylistic-quirks.md`「ひらがな書き選好」で最終稿に「かなのまま残る」語は、推敲でのかな→漢字化が徹底されなかった残存物である可能性があり、開く／閉じるの逆方向の癖として因果を取り違えない。

### 精度・妥当性の微調整（範囲限定・断定緩和・強調語削除）

- 対象：AI草稿の主張の範囲・程度・主体（CG基礎の草稿→完成差分）。
- 修正前の特徴：過剰な強調語や、範囲・主体が広すぎる断定が含まれる。
- 修正後の傾向：範囲を狭める語や限定語を補い、過剰な強調語を削り、断定的な能動表現を控えめにする。
- 修正操作：強調語の削除（「ちょうど逆向き」→「逆向き」）、断定の緩和（「処理を行うこともできます」→「処理に使われることもあります」）、範囲限定（「役割」→「一分野」）、具体化・限定の付加（「画像生成」→「CGによる画像生成」、「カメラのレンズへ」→「どれだけの光がカメラのレンズへ」）。
- 修正の目的：主張の範囲・程度・主体を正確に限定する。
- 差分の種類：表現の書き換え／説明の粒度調整。
- 適用する状況：AI草稿の主張が過剰に強い・広い箇所。
- 適用しない状況：既に十分限定された表現、意図的に断定してよい定義文。
- 確度：条件付きの傾向（強い推測）。CG基礎2記事に分布する（CGと画像処理6件、カメラでの撮影とCG2件）。ただし「精度調整」という括りは性質の異なる微修正の束ねであり（強調語削除は実質「ちょうど」1例に依存）、統一ラベルは推測にすぎない。各サブ操作は1〜2件と薄く集約でのみ成立する。「反射するか」→「反射するか？の計算結果」や「どれだけの光が」の付加は主張強度の調整というより技術的明確化寄りで、文体修正とは分けて考える（技術的正確化を文体修正として扱わない）。
- 根拠：CGと画像処理（087d64c→3e63a05）「役割」→「一分野」、「ちょうど逆向き」→「逆向き」、「行うこともできます」→「使われることもあります」、「画像生成の材料」→「CGによる画像生成の材料」。カメラでの撮影とCG（50cac82→37f6153）「カメラのレンズへ」→「どれだけの光がカメラのレンズへ」。
- 反例・例外：すべての強調語を機械的に削るわけではない。誇張の抑制・限定語付加は一般的な編集術とも重なり、著者固有性は限定的。
- 関連ルール：完成本文でのヘッジ・限定の言い回しは `stylistic-quirks.md`「ヘッジ文末」「限定・留保の言い回し」。表現の癖（完成本文）と差分（推敲操作）は別階層。

### 常体（である調）の下書きを敬体（ですます調）へ整える

- 対象：常体・体言止め・句点なしで書かれた下書きの地の文。
- 修正前の特徴：下書きが常体（〜である／〜という／〜呼ぶ／体言止め・句点なし）で書かれている。
- 修正後の傾向：地の文の文末を敬体（〜です／〜します／〜といいます／〜呼ばれます）へ統一し、欠けていた句点を補う。
- 修正操作：常体→敬体の変換、句点の補充。
- 修正の目的：完成記事のレジスター（敬体）へ揃える。
- 差分の種類：表現の書き換え。
- 適用する状況：下書きが常体・句点なしの本文プローズの場合。
- 適用しない状況：下書きが既に敬体、または箇条書きメモの場合（地の文化する際に最初から敬体で書かれる）。bullet・番号リストには及ばず、常体・体言止めが残る場合がある。敬体化は本文プローズに限る。
- 確度：条件付きの傾向（直接確認できる事実）。眼の構造・照明と照明光・照度と照明設計の3記事の推敲差分で確認でき、従来の「単一記事のみ」からは補強された。ただし純粋な文体調整のみのコミットは眼の構造（b8cde7e→d3386e4、コミット題「文体を整えて仕上げ」）と照明と照明光（76f2035「文体の調整」）で、照度と照明設計（de98e14）は内容追加・再構成を伴う複合コミットのため単独では弱い。だ・である→です・ます化自体は技術記事一般に見られる汎用的な推敲作法でもあり、著者固有の癖と断ずるには含みがある。「統一」は地の文限定で、bullet・番号リスト（眼の構造の三層構造・屈折リスト）や未推敲の箇条書きセクション（照度と照明設計）は常体のまま残る。
- 根拠：
  - 眼の構造（b8cde7e→d3386e4）：ほぼ全文で〜という/〜である/〜する→〜といいます/〜です/〜します。
  - 照明と照明光（76f2035）：「照明という」→「照明といいます。」、「呼ぶ」→「呼ばれます」、「分けられる」→「分けられます。」（句点補充を伴う）、「である」→「ですが」。
  - 照度と照明設計（de98e14）：推敲済みの照度セクションはです・ます体の連文（「指標が照度です。」「表します。」「なります。」）、未推敲の照明設計セクションは箇条書き＋だ・である体（「規定されている」）のまま残る対比。
- 反例・例外：箇条書きメモの下書きでは常体→敬体の「変換」としては現れない（完成形は全記事が敬体で一致）。照明と照明光の後続コミット ef3c1c9 では接続助詞「し」→「ており」の微調整や mermaid ラベルの語尾（「白熱電球」↔「白熱電球による光」）が往復しており、細部の言い回し・接続表現は安定しない。
- 関連ルール：完成本文の敬体レジスターの事実は `stylistic-quirks.md`「やさしい指南役の丁寧語」。ここでは変換操作を扱う。

## 追加する傾向

### 理由・背景の補足

- 対象：下書き段階で事実だけを述べていた記述。
- 修正前の特徴：用語の命名や現象が、理由・因果なしに事実として置かれている。
- 修正後の傾向：命名理由や現象の因果を説明する段落を、結論の後ろに補う。
- 修正操作：理由・背景の追記。
- 修正の目的：読者が名前・現象を理由づけで捉えられるようにする。
- 差分の種類：情報の追加。
- 適用する状況：概念解説で、命名や現象の理由を添えられる箇所。
- 適用しない状況：手順そのものを述べるチュートリアルの各ステップには理由補足を付けない傾向。理由づけを伴わず事実だけを述べる節も多く、全節に補うわけではない。
- 確度：条件付きの傾向（強い推測）。
- 根拠：
  - マンセル表色系（3aa9ba3 → a4b2b8c）：「カラーツリー」の命名理由（樹木のように成長する可能性）を追記（既存記述の書き換え・拡張）。
  - PCCSによる色表示（1b66b9d → d8d6c80）：色立体が斜めに歪む理由（色相ごとの純色の明度差）をデモの前に新規追記。
  - 視覚・色覚シリーズ（眼の構造 b8cde7e→d3386e4／反対色応答 7012163→7f63600／色覚説 c95ef39→edf7738／明るさ efc1f6c→c44ad78）：仕組み・部位を並べた記述に「このように瞳孔の大きさを変えることで、眼に入る光の量を調節しています」「円形の孔であるため、特に暗く見えるのです」など、なぜ・何のためかの補足文を追加する。眼の構造では草稿が既に地の文で、そこへ補足が新規追加されており、箇条書き→地の文化の副産物ではなく独立した補足追加操作と確認できる。追加の意図（読者の理解を助ける）は推測。文末「〜のです」「〜ため〜」は `stylistic-quirks.md`「因果は「〜ため（に）」優先、後置理由は「〜からです」」の担当で、ここでは追記操作のみを扱う。
  - 混色シリーズ（等色実験とグラスマンの法則 28f0024／実世界でみられる混色 5b7219f）：推敲時に、読者がつまずきやすい記号・用語・「なぜ」への補足を後から足す。等色実験では $$[R],[G],[B]$$ や $$R,G,B$$ の表記慣習を説明する短い記号補足の `:::Note` を追加する（コミット題も「記号についての補足を追加」）。実世界では顔料／染料の定義を `:::Note` で足し、さらに「### 黒のインクが必要な理由」という小見出しを新設して C・M・Y だけで黒を作れない理由を補足する。記号補足の Note は初出行の直後ではなく、その記号を正式に定義・命名する箇所に置かれる点に注意する（例：$$[R]$$ の初出より後の、三色性を言い換える・三刺激値と呼ぶ箇所）。概念的な疑問に独立見出しを立てるのは実世界の黒インク1例のみ（n=1）で弱く扱う。両記事とも同一シリーズ・同時期のため確度は上げない。
  - CG・幾何変換シリーズ：結論や比喩だけで済ませた箇所に「なぜその形になるか」を示す途中式・機構の説明・数値／幾何の具体像を追記して深掘りする（変換の合成 f3b00d7＝非可換性を実際の行列積 RT/TR と幾何的な動きで例示、基本的な変換 4840d2e＝同次座標を一般行列の導出で動機づけ・0d7466e＝w=0 を2点の差から導出、鏡映 70d68da＝y=x鏡映を行列係数の決定手順で導出、アフィン 3efe69f・40bdec1＝2枚の鏡による平行移動2d・定数倍同値・無限遠点の導出を追加）。人手の操作は「どこを深掘りするか」の判断と配置であり、追記本文の多く（合成・鏡映・アフィン）は人手起筆・一部（基本的な変換）はAI再生成と混在する。いずれにせよ追記された文面の言い回しは記事ごとに揺れるため著者の癖としては一般化しない。
  - 塗りつぶし処理（自己下書き 16758df→78ba48c）：交点が偶数個になるという断定「交点はふつう偶数個できます」の後に、スキャンラインを左端からたどると外と内を交互に行き来し、その回数の分だけ交点が偶数個そろう、という理由を述べる段落を新設する。
- 反例・例外：理由づけを伴わない節も多い。冗長・重複した補足はむしろ削られる（→「冗長・重複した説明や過剰な例えを削る」）ため、補足なら何でも足すわけではない。
- 関連ルール：完成記事における「命名の由来を添える」判断は `thinking-flow.md`「命名の理由づけ」。完成記事からの推測（思考）と、差分で確認できる追記操作（推敲）は別階層。

- 適用する状況：概念解説で、表記法や分類の抽象的な説明があるとき。
- 適用しない状況：すでに十分具体的な内容には足さない。
- 確度：条件付きの傾向（直接確認できる事実）。根拠はマンセル表色系と塗りつぶし処理（CG基礎の描画）で、記事タイプをまたいで確認できる。ただし数値例と日常場面を1操作に束ねている点、一方の根拠が初稿執筆に近い点に留意する。抽象を具体で裏づける具体化自体は一般的な編集術で、著者固有性は限定的。
- 根拠：
  - マンセル表色系（3aa9ba3 → a4b2b8c）：HV/C と N の記法に `:::Example` の数値例（5R 3/8、N5）を追加、青系のPCCSとのずれに 5B の実例を追加。
  - 塗りつぶし処理（自己下書き 16758df→78ba48c）：スキャンラインと閉領域の交点が偶数個になる仕組みを、「山が `2` つ並んだ形」に写した :::Example として新設する。
- 反例・例外：例を伴わず定義だけで済ませる節もある。
- 関連ルール：完成記事における「規則・図の直後に代表例を置く」配置は `writing-style.md`「例の配置」。

### 節・記事の冒頭に見通し・誘導の導入を追加する

- 対象：本題の説明の前。
- 修正前の特徴：見通しや誘導なしにいきなり本題へ入る。
- 修正後の傾向：これから述べる内容の要約や見通しを記事・節の冒頭へ追加する。「ここでは〜を考えてみましょう」などの誘導や読者への問いかけを伴うことがある。
- 修正操作：導入（概観・誘導・問いかけ）の追加。
- 修正の目的：本題へ入る前に見通しを与える。
- 差分の種類：情報の追加。
- 適用する状況：長い節や記事全体の冒頭。
- 適用しない状況：既に文脈が明らかな短い節には追加しない。
- 確度：条件付きの傾向（直接確認できる事実）。反対色応答（最初の見出し前に反対色応答3信号の概観を追加）と明るさ（「ここでは、明暗の変化への順応について考えてみましょう」「では、薄暗い場所では…働くのでしょうか？」）で確認。誘導/問いかけ表現は明るさに集中。冒頭に導入を置くこと自体は一般的な文章術。
- 根拠：反対色応答と色の識別（7012163→7f63600）、明るさに対する感度と順応（efc1f6c→c44ad78）。
- 反例・例外：眼の構造の節冒頭「黒目を見ると、真ん中に特に色が暗い部分があります」は観察的なリード文で、要約・見通しの追加としては fit が弱い。
- 関連ルール：誘導・問いかけ表現（〜てみましょう／〜でしょうか）は `stylistic-quirks.md`「誘い掛け」「節頭の反語的な問いかけ」。

### 前後のセクション・関連記事への参照と誘導を追加する

- 対象：関連記事・前後の節への参照。
- 修正前の特徴：他記事・後続節への参照や誘導がない。
- 修正後の傾向：関連する他記事へのリンクや「次のセクションで学ぶ」など前後の節への参照・誘導文を追加する。
- 修正操作：参照リンク・誘導文の追加。
- 修正の目的：シリーズ内の位置づけと導線を補う。
- 差分の種類：情報の追加。
- 適用する状況：相互リンクされたシリーズ内の記事。
- 適用しない状況：独立して完結する短い記述。相互リンクされた知識ベースという媒体構造に依存する面が大きい。
- 確度：条件付きの傾向（直接確認できる事実）。色覚説（「その詳細については、次のセクションで学ぶことにしましょう」＋加法混色リンク）と反対色応答（段階説／加法混色／分光視感効率曲線への複数リンク新規追加）の2記事で確認。混色シリーズの実世界でみられる混色（5b7219f）でも、冒頭に「[加法混色と減法混色]や[さまざまな加法混色]で学んだ混色原理の組み合わせによって…」という既習リンク付きの導入文を後から追加し、記事をシリーズ内に位置づけている。ただし混色シリーズでこの「冒頭に既習リンク導入を後から足す」実例は1記事のみ（n=1）で、前提記事へのリンク自体はコーパス全体に見られる確立した相互参照の習慣でもあるため、確度は上げない。
- 根拠：色覚説（c95ef39→edf7738）、反対色応答と色の識別（7012163→7f63600）、実世界でみられる混色（d85f930→5b7219f）。
- 反例・例外：眼の構造では他記事参照の追加は見られない。前提記事のないシリーズ先頭の記事には当てはまらない。
- 関連ルール：完成本文で既出概念をリンクへ委ねる配置は `writing-style.md`「相互参照：既出概念はリンクに委ねる」。

## 削除する傾向

CG基礎シリーズは実際のAI草稿→人手編集差分が得られるため、AIらしいメタ足場の除去を確認できる（色の理論シリーズには該当差分がない）。

### AI草稿のメタ足場・自己言及・冗長な再要約の除去

- 対象：AI草稿が挿入するメタ足場・自己言及・冗長な再要約。
- 修正前の特徴：(1)「このページでは次の◯つの側面から見ていきます」＋予告の箇条書き、(2)末尾の「ここまで見てきたように…このあとのページでは…話を進めていきます」というまとめ＋次ページ予告の Note、(3)「この記事では」「前のページでは…見てきました」等の記事構造への自己言及・位置参照、(4)直前ページ内容の冗長な再要約や既出用語の再定義、(5)「ここが重要な違いです」「しかし、ここで見落としてはならない大切な事実があります」のような煽り気味の強調・転換文、「あくまで画像を生み出すための設計図であり」「この一見地味な処理こそ〜橋渡しする要となります」のような比喩過多の再述、「前の節で見た〜が、まさにこのラスタ化です」のような節間の自己参照ポインタ。
- 修正後の傾向：要点だけ残して削除・圧縮する。予告リストと末尾の総括＋次ページ予告 Note は丸ごと削除し、前ページ recap は1文へ圧縮する。リンクテキスト「前のページ」はリンク先の概念名（「デジタルカメラモデル」）へ置換する。
- 修正操作：メタ足場の削除・圧縮、自己言及の中立化、リンクテキストの概念名化。
- 修正の目的：AIらしい足場を外し、読者に必要な情報だけを残す。
- 差分の種類：情報の削除／構成の変更。
- 適用する状況：AI草稿の概念解説を本稿へ仕上げるとき。
- 適用しない状況：導入で全体像を一文で予告する程度の記述（「その全体像を…見ていきます」は残す）、読者ナビゲーションとして本当に必要なリンク。
- 確度：条件付きの傾向（直接確認できる事実）。従来はCGと画像処理の1記事に依存していたが、デジタル画像シリーズの複数記事（デジタル化・量子化・カラー画像）で予告文・次トピック予告・末尾総括＋次ページ誘導・冗長な recap の削除/圧縮が反復確認でき、傾向として補強された。さらに景観色彩の考え方・住宅の色彩設計プロセス（色の活用分野）でも、節頭の「これから〜を見ていきます」式の予告文や「いよいよ」「こうした〜が土台になる」といったつなぎ・地ならし文の削除を確認でき、CG系・同一時期という偏りは緩和された（テーマ・時期の異なる系統で反復）。(5) の修辞的強調・比喩・自己参照ポインタの削除はベクタ表現とラスタ表現1記事に集中し（他記事は各1箇所と薄い）、単独では弱い傾向にとどまる。グラデーション生成（もう一つのCG基礎AI草稿）でも予告文・冗長な対比/定義の圧縮・情緒過多の削除を確認できるが、同一シリーズの自己下書きでは軽量な予告文が温存されるため、予告文の削除は機械的な一般則にはしない（削除対象は冗長・重複・煽り・情緒過多に寄る）。
- 根拠：CGと画像処理（087d64c→3e63a05）：導入の「次の3つの側面から見ていきます」＋3項目リストを削除、前ページ recap と「画像処理とは…技術です」の再定義を1文に圧縮、リンクテキスト「前のページ」→「デジタルカメラモデル」、末尾の総括＋次ページ予告 :::Note を丸ごと削除。カメラでの撮影とCG（50cac82→37f6153）：「この記事では」→「ここでは」。デジタル画像シリーズ：画像のデジタル化（6686939→d667dde）で導入「ここでは…全体像を見ていきます」・標本化節末「次は…見ていきます」・末尾「こうして…分かりました。では…別のページで詳しく見ていきます」＋TODO を削除、画像の量子化（72ef77d→f8e815c）で導入 recap を1文へ圧縮し末尾 forward-ref を削除、カラー画像とデータ量（61a1bc2→a7f1307）で導入予告と遷移文を削除。(5) はベクタ表現とラスタ表現（f6a53ff→8e705e2）で「ここが重要な違いです」「しかし、ここで見落としてはならない…」「あくまで…設計図であり」「この一見地味な処理こそ…要となります」「前の節で見た…まさにこのラスタ化です」を削除し、CG本質の冗長段落を1文へ圧縮。CG・幾何変換シリーズでも、冒頭や節頭の「〜を整理しておきましょう」「まずはそれぞれを順番に見ていきます」「ここでは代表的な2つを取り上げます」「個別の変換の総まとめとなる節です」のような予告メタ文・節の自己説明を削除して定義・本題から直接書き始め、導入での過剰な既習リンクの列挙も1〜2本へ間引く（座標系 42f5a61、基本的な変換と同次座標 6ac887e、アフィン変換 3318224、鏡映）。ただし予告メタは削る一方で記事末尾の前向きな誘導は残す・むしろ足す非対称があり、基本的な変換 c916324 では「次は、このような変換の合成について学びましょう」を追加している。グラデーション生成（AI草稿 65bcd5b→0aec308）：導入末の予告文「このページでは…グラデーションをどのように生成するのかを見ていきます」と旧『濃淡処理』節末の次トピック予告「ここで必要になるのが、次に見る補間の考え方です」を削除し、あわせて『濃淡処理』見出しを削除して本文を導入直下へ畳み、導入で既に述べた塗りつぶしとの対比の再説明（「塗りつぶしが『どこを塗るか』…濃淡処理では…」）と定義の言い換え（「これは、位置に応じて…処理のことです」）を削って `5` 文→`2` 文へ圧縮する。同記事の導入 `2` 文目の情緒的な効能アピール「奥行きや立体感、やわらかな雰囲気を生み出します」も削除される。景観色彩の考え方（3468898→fa830fa）では「それぞれの色彩が、どのような性格をもっているのかを見ていきます。」の予告文と「このように、景観をつくる要素は住宅や建物にとどまりません」の総括前置きを削除、住宅の色彩設計プロセス（80c7fcb→8d4f0b7）では「周辺の環境をつかんだら、いよいよ具体的な色を決めていきます。」「こうした流れの土台になるのが、家の建つ場所の周辺環境を知ることです。」の地ならし文を削除する。
- 反例・例外：カメラでの撮影とCGは導入の「その全体像を…見ていきます」という予告文を保持しており、軽量な見通し一文までは削らない。CG・幾何変換シリーズでも記事末尾の前向きな誘導文は削らず残す／足す。画像の標本化は末尾のリンクを整理するがトピック予告文自体は保持し（予告を機械的に削らない反例）、ベクタ表現とラスタ表現の末尾の姉妹ユニット参照も保持される。導入で全体像を一文だけ示す軽い見通し文や、読者ナビゲーションに必要な相互リンクは残す。グラデーション生成での予告文削除も一般則ではなく、軽量な予告文（「〜を見ていきます」「その土台となるのが、次に見る〜です」）はむしろ温存される傾向が強い。同シリーズの塗りつぶし処理（自己下書き 16758df→78ba48c）は、導入「ここでは、ある領域の内側を色で埋める塗りつぶし処理について見ていきます」と節末「その土台となるのが、次に見る連結性です」を一字一句変えずに残す（グラデーション生成での予告文削除が著者の一般的な選好ではないことを示す反例）。情緒表現も一律には排除せず、グラデーション生成の補間関数節「片側に余韻を残したい」は保持される。冗長を削る一方で、不足していた具体説明（`t` の求め方など）はむしろ追加される（→「数値例・具体場面の追加」「曖昧・一般的な記述を文脈に応じて具体化する」）。景観・住宅シリーズでも、定義や要点を担う導入の一文は削らず、内容の薄い予告・接続だけを対象とする。
- 関連ルール：後続記事で前提を短く再活性化する判断は `pending/thinking-flow.md`（保留）、その配置は `writing-style.md`「シリーズ内位置による導入の変化」。実質的な用語定義の Note 追加（→ `pending/refine-style.md`「AI草稿の未説明用語への定義・補足の追加」）とは対象 Note の内容が異なるため矛盾しない。

### 冗長・重複した説明や過剰な例えを削る（自己推敲）

- 対象：同趣旨を繰り返す箇条書き、説明の助けにならない冗長な例え話（視覚・色覚シリーズの自己改稿）。
- 修正前の特徴：同じ趣旨の重複した箇条書きや、的外れな例えがある。
- 修正後の傾向：簡潔な一文へ統合するか削除する。
- 修正操作：重複の統合・過剰な例えの削除。
- 修正の目的：冗長・重複・的外れを除いて簡潔にする。
- 差分の種類：情報の削除。
- 適用する状況：冗長・重複・的外れな例えがある箇所。
- 適用しない状況：情報として必要な補足や、理解を助ける具体例には適用しない（因果・目的の補足はむしろ追加される→「理由・背景の補足」）。
- 確度：条件付きの傾向（直接確認できる事実）。反対色応答で「引き算で共通部分が消える」趣旨の重複した箇条書きを一文へ統合、明るさで「100点満点のテスト」に例える Note を削除し「分光視感効率曲線の最大値は1.0です」へ簡素化。デジタル画像シリーズでも、画像の標本化はモアレの数値 Example（「1mmに10本の縞…2mmおきの標本点…」）を直前段落と重複するため削除し（642ebc5）、標本化定理の語源 Note（「正式にはナイキスト-シャノン標本化定理…名にちなむ」）を削除する（dd09e46「余分な情報を削除」）。カラー画像とデータ量は 1920×1080＝6.2MB の詳細ワークト Example を削除して一般式へ置換する（a7f1307、この削除は `pending/refine-style.md`「詳細な数値ワークト例を一般式へ置換する（保留）」と同一根拠のため二重計上しない）。自己推敲全体としては加筆が主で削除は少数。CG・幾何変換シリーズでも、記事末尾に置かれた網羅的な比較まとめ（既出情報の再掲）、本筋から外れる補足 Note、「とても重要です」のような煽り気味の強調文を削除する（アフィン変換 3e4aeb7＝「変換の比較まとめ」節を丸ごと削除・74ec9e1＝アフィン不変量の Note 削除、変換の合成＝画像処理向け Note 削除、座標系 42f5a61＝「…とても重要です」の強調削除、鏡映＝スキュー＝合成分解の Note 削除・用途節を大幅縮約）。ただし本筋の理解に必要な補足はむしろ Note として残す・新設する（基本的な変換で原点中心の注記を Note 化、鏡映で係数 a=tanθ を Note 化）ため、削除対象は傍論・重複・煽りに限る。
- 根拠：反対色応答と色の識別（7012163→7f63600）、明るさに対する感度と順応（efc1f6c→c44ad78）、画像の標本化（642ebc5・dd09e46）、カラー画像とデータ量（a7f1307）、CG・幾何変換シリーズ（アフィン 3e4aeb7・74ec9e1／変換の合成／座標系 42f5a61／鏡映）。
- 反例・例外：明確な「削除」として独立するのは明るさの過剰な例え削除1件で、重複bullet統合は「箇条書き→散文」に付随して生じる部分が大きい（二重計上しない）。理解を助ける Example（擬似輪郭の空グラデーション例、ラスタ表現のスマホ拡大例）は保持されており、例を一律に削るわけではない。
- 関連ルール：因果・目的の補足はむしろ追加される非対称は本ファイル「理由・背景の補足」。AI草稿由来のメタ足場除去（CG基礎）は「AI草稿のメタ足場・自己言及・冗長な再要約の除去」。

### 冗長な理由づけ・敷衍を圧縮する（物理的・機能的制約の理由は残す）

- 対象：AI草稿が要点の後ろに足す「〜ためです」式の理由説明や、同義の言い換えによる敷衍。
- 修正前の特徴：要点の直後に、装飾的・情緒的な理由づけ（素材へのなじみやすさ等）や、比較の便を述べる敷衍が続く。
- 修正後の傾向：薄い理由づけ・敷衍を削り、要点だけを短く残して情報密度を上げる。AIが膨らませた説明的な散文を、元メモに近い簡潔さへ引き戻す方向で編集する。
- 修正操作：理由節・敷衍の削除・圧縮。
- 修正の目的：要点主体の簡潔さへ引き戻す。
- 差分の種類：情報の削除／説明の粒度調整。
- 適用する状況：AI草稿の概念解説で、要点に薄い理由づけ・敷衍が付いている箇所。
- 適用しない状況：技術的な前提や例外の但し書き（面積効果で暗色は逆に暗く見える等）は残す。また、物理的・機能的制約に基づく理由づけ（耐候性・退色・太陽光や雨風などの環境要因）は削らず残す。削るのは装飾的・敷衍的な理由づけに限る。常に短くするのではなく、要る内容は TermCard 等へ構造化して残す使い分けがある。
- 確度：条件付きの傾向（直接確認できる事実）。根拠は住宅の色彩設計プロセス1記事に閉じ、冗長圧縮自体は一般的な編集術とも重なる。なお同編集の差分行数の多くは著者自身の元メモ（HTMLコメント）削除によるもので、AI散文の圧縮量とは切り分ける（差分の増減行数は圧縮の根拠にしない）。
- 根拠：住宅の色彩設計プロセス（80c7fcb→8d4f0b7）：「落ち着いた暖色系の低彩度色は、土や木といった…なじみやすいためです。」を削除して「暖色系の低彩度色をベースに検討することもあります」へ短縮、「景観に関する色彩の基準がマンセル値で表されることが多く、周辺の色と比べやすいためです。」を削除。
- 反例・例外：住宅の街並み配色の節では逆に TermCard で説明を足すなど、常に短縮するわけではない。物理的・機能的な理由づけ・技術的な但し書きは残す。
- 関連ルール：一般的な重複・過剰な例えの削除は本ファイル「冗長・重複した説明や過剰な例えを削る（自己推敲）」。削る側（装飾的な理由づけ）と残す側（物理的・機能的制約の理由づけ）の対は、規範を裸で示さず物理的理由で根拠づける判断（`thinking-flow.md` の環境的・物理的な理由づけに関するルール）と分担する。

### 曖昧な前方・後方参照はリンク化するか、未執筆・シリーズ外なら削除する

- 対象：「後の記事で扱います」「別の記事で改めて取り上げます」「後続の〜セクションで詳しく扱う」のような、参照先が曖昧な前方・後方参照。
- 修正前の特徴：参照先が具体的に示されず、実在するとも未執筆とも判別できない曖昧な参照が置かれている。
- 修正後の傾向：(a) 参照先がシリーズ内に実在するなら `[変換の合成と順序](...)` 等の具体的な内部リンクへ置き換える。(b) 参照先が本シリーズ外・未執筆なら文・段落ごと削除する。記事末尾には次に読む記事への内部リンクを置いて締めることがある。
- 修正操作：曖昧参照のリンク化、または未執筆・シリーズ外参照の削除。
- 修正の目的：読者が実際に辿れる導線だけを残し、辿れない約束を消す。
- 差分の種類：表現の書き換え／情報の削除。
- 適用する状況：シリーズ内の記事で曖昧な参照があるとき。
- 適用しない状況：参照先がシリーズ内に実在して読者が辿れる場合は削除せずリンク化する。
- 確度：条件付きの傾向（直接確認できる事実）。基本的な変換と同次座標 c4dc96c（「後の記事で扱います」→「合成変換の章で扱います」、Note「別の記事で」→[射影変換]リンク）、アフィン変換（消点「後続のセクションで改めて登場します」・透視投影「後続の投影のセクションで詳しく扱います」・ホモグラフィの「後続のイメージモザイキング…で扱います」段落を削除）、鏡映とスキュー 49333e1（末尾に[アフィン変換と射影変換]リンク追加）、変換の合成（後方参照「ここまでに見た拡大縮小や回転」→[基本的な変換と同次座標]リンク）で確認。前方・後方いずれの曖昧参照にも及ぶ。根拠はCG・幾何変換の単一シリーズに閉じる。
- 反例・例外：参照先が実在する場合は削除ではなくリンク化になる（削除と置換の2方向がある）。
- 関連ルール：関連記事への参照・誘導を追加する近縁の操作は本ファイル「前後のセクション・関連記事への参照と誘導を追加する」。完成本文で既出概念をリンクへ委ねる配置は `writing-style.md`「相互参照：既出概念はリンクに委ねる」。

## マークアップの正規化

### 未マークのキー概念語への :Mark 付与（編集判断）

- 対象：AI草稿で :Mark[] が付いていない、本文でキー概念として提示すべき語（CG基礎の草稿→完成差分）。
- 修正前の特徴：キー概念語が :Mark 未付与のまま、または並列・対比・まとめ箇所での反復言及で付与が不揃い。
- 修正後の傾向：未マークのキー概念語に :Mark を付与する。まとめ・対比・箇条書きでの反復言及にも、並列箇所の整合・強調のため付与する。
- 修正操作：:Mark の付与。
- 修正の目的：どの語をキー用語として提示するかを揃える編集判断。
- 差分の種類：表現の書き換え（マークアップ層）。整形寄りの操作でもある。
- 適用する状況：AI草稿のキー概念語が未マーク、または反復箇所で付与が不揃いなとき。
- 適用しない状況：既に :Mark 済みの語、キー概念として提示しない一般語。
- 確度：条件付きの傾向（直接確認できる事実）。CG基礎2記事では計7〜8箇所だったが、デジタル画像シリーズ5本の草稿→完成差分では数十箇所に拡大し、量・一貫性ともに補強された。まとめ／対比／箇条書きでの反復言及への付与（既に別箇所で :Mark 済みの語の並列箇所への整合付与）が相当数を占め、「初出」だけでは説明できない。正しくは「未マークのキー概念語への付与（反復箇所の整合を含む）」。CG基礎（描画）シリーズの2記事でも同種の付与を確認でき量的には補強されるが、キーワード強調は編集フローの定型作業・記法慣習の側面が強く、著者固有の癖とは言い切れない。根拠は依然CG系の同一シリーズ・同一時期に偏る。
- 根拠：カメラでの撮影とCG（50cac82→37f6153）：画像／レンズの焦点距離／CG／画像処理 に :Mark を追加。CGと画像処理（087d64c→3e63a05）：CG／画像処理／視覚／コンピュータビジョン（箇条書きでの反復）に :Mark を追加。デジタル画像シリーズ：画像のデジタル化（アナログ画像／デジタル化／標本化／量子化／符号化／画素／ピクセル）、標本化（空間周波数／標本化間隔／2倍／高／低／エイリアス／標本化定理）、量子化（階段関数／ビット数／2倍／8ビット量子化／量子化誤差／半分／量子化雑音／擬似輪郭）、カラー画像（グレースケール画像／カラー画像／チャンネル／255／3倍）、ベクタ/ラスタ（ラスタ表現）に :Mark を付与。初出だけでなくまとめ・対比・並列箇所の反復言及にも整合のため付与し、数字・記号（`0`/`1`、`2倍`、`255`、`半分`）も強調対象なら :Mark で示す。CG・幾何変換シリーズでも、平文のままだった各節の鍵語・方向語・概念語に :Mark[] を付与する（座標系＝座標/右/上/下/角度/回転/傾き、基本的な変換＝平行移動/拡大・縮小/回転/足し算/対角線/方向ベクトル、変換の合成＝左/逆行列、鏡映＝上下/左右/非対角成分、アフィン＝直線性/平行性/遠近感/透視投影/無限遠点/消点）。ただし同語のすべての出現に付けるのではなく節ごとの中心語・初出に限り、アフィンでは一部の :Mark を平文へ戻す逆方向の編集もあるため、全出現への機械的付与ではない。グラデーション生成（AI草稿 0aec308→5128584）と塗りつぶし処理（自己下書き 16758df→78ba48c）でも同様に、補間関数節の 直線／関数／2次関数／対数関数、2次元節の 2次元／平面／方向／放射／角度、ディフュージョンカーブ節の 拡散、および スキャンライン／シードフィル／偶数 などのキー概念語へ編集時に :Mark を付与する。
- 反例・例外：導入の「画像」はキー概念というより一般語寄りで、判断の境界例。CG・幾何変換シリーズでも同一語の2回目以降や強調の焦点でない語には付けず、一部は平文へ戻す。:Mark 付与自体は散文の文体修正ではなくマークアップ層の整形操作であり、修正と整形の境界にある。
- 関連ルール：:Mark の記法・付与書式そのものは `syntax-guide.md`（記法・書式の正）が担当。どの語をキー用語・要点と判断するか（程度語・要の数値まで含める選定基準）は `stylistic-quirks.md`「強調（:Mark）の対象に程度語・要の数値まで含める」を参照し、ここでは未付与→付与・反復箇所の整合という編集操作を扱う。

## 技術説明を修正する傾向

技術的な正確さ・記法の整形にあたる修正で、文章の文体修正とは分けて扱う（出力契約「技術的な誤りの訂正を文体修正として扱わない」に従う）。根拠は主にCG・幾何変換シリーズに閉じるが、一部はCG基礎（描画）シリーズでも確認できる。

### 数式・変数の表記を機械的に統一する（技術的整形）

- 対象：シリーズ内で揺れる数式・変数の表記。
- 修正前の特徴：添字・プライム・括弧・数式化の記法が記事間／記事内で揃っていない。
- 修正後の傾向：(a) tx,ty,sx,sy → t_x, s_x 等の下付き添字、(b) x' → x^{\prime} のプライム記号、(c) 地の文中の単一変数を inline code から $$...$$ 数式へ、(d) 行ベクトルの角括弧 [0 0 1] → 丸括弧 (0 0 1)、(e) 係数記号の統一（スキュー係数 b→a）へそろえる。
- 修正操作：記法の機械的統一。
- 修正の目的：数式表記をシリーズで一貫させる。
- 差分の種類：整形・記法統一（文体修正とは区別）。
- 適用する状況：数式を伴う実装解説。
- 適用しない状況：図解中心で数式のない箇所。
- 確度：条件付きの傾向（直接確認できる事実）。基本的な変換と同次座標 454a720（添字）・1c659a5（プライム）・c916324（[0 0 1]→(0 0 1)）、変換の合成と順序（v/T/R/S を inline code→$$...$$）、鏡映とスキュー（スキュー係数 b→a）で確認。ただし完全には一貫せず、アフィン変換では一部 $$w^{\prime}$$ を inline `w'` に戻す調整もあり（HEAD で `w'` と w^{\prime} が混在）、方向は概ね「添字・プライム・丸括弧・数式化」だが例外がある。
- 反例・例外：アフィン変換の $$w^{\prime}$$→`w'` 戻しのように方向が逆の調整もある。
- 関連ルール：完成本文でのCG数値のインラインコード化は `stylistic-quirks.md`「数量・ビット値・2進リテラルをインラインコードで表記する」（記法規約の正は `syntax-guide.md`）。

### 語の厳密性・説明のスコープを補正する

- 対象：AI草稿の用語・スコープが不正確・広すぎる箇所。
- 修正前の特徴：語の厳密性やスコープが甘い（軸／直線の混同、距離が負値を許さない、回転定義が原点前提、剛体変換の分類位置、難解語）。
- 修正後の傾向：語の厳密性・スコープを上げる。鏡映の対称線を「軸」→「直線」（座標軸との混同回避）、座標成分の「距離」→「移動量／進んだ量」（負値許容）、回転の定義文から「原点のまわり」を外し導出時に原点を明示、剛体変換をユークリッド変換の別名として分類の入れ子を修正、「透視短縮」→「遠近感」へ置換。
- 修正操作：用語・スコープの技術的訂正。
- 修正の目的：技術的な正確さとスコープを正す。
- 差分の種類：技術的訂正（文体修正とは区別）。
- 適用する状況：概念解説・実装解説で語やスコープが不正確な箇所。
- 適用しない状況：誤字・リンク・整形など純粋な機械的訂正はここに含めない（別扱い）。
- 確度：条件付きの傾向（強い推測）。鏡映（軸→直線）、座標系 42f5a61（距離→移動量）、基本的な変換（回転定義のスコープ調整）、アフィン ba7b92a（剛体変換＝ユークリッド変換の同義化・分類修正）、透視短縮→遠近感で確認。ただし個々の訂正は記事テーマ固有の内容判断であり、そのまま他テーマへは一般化できない（操作の型として抽出）。剛体＝ユークリッド同義化は標準的定義（剛体＝向き保存の狭義）とは異なる著者の編集判断であり、「技術的正確さの向上」と断ずるには含みがある。グラデーション生成 5128584 では「3次元曲面として捉える」→「3次元曲面に生じる陰影として捉える」と、比喩の指す対象（曲面そのものではなく、そこに生じる陰影）を正確化する。
- 関連ルール：AI草稿の主張の範囲・程度を正確に限定する近縁の操作は本ファイル「精度・妥当性の微調整（範囲限定・断定緩和・強調語削除）」。
