# 明度マッチング カードゲーム（lightness-match）

ある基準色を表示し、それと「明度（マンセルValue）が同じ」カードを候補の中から探す選択式のカードゲーム。色相・彩度に惑わされず明度だけを見抜く知覚トレーニングを、気軽に遊べる形で提供する。

## 目的・背景

- **彩度と切り分けて「明度」だけを見抜く知覚トレーニング**を、気軽に遊べるゲームの形で提供する。
- 想定プレイヤーは、色彩を学習中の人の練習と、サイト来訪者のカジュアルな遊びの両方。
- 「同じ明度でも鮮やかな色ほど明るく見える」といった錯覚に流されず、明度を独立した軸として捉えられるようにすることが狙い。

## スコープ

### やること

- 選択式のカードゲーム 1 種。
- 難易度 2 段（**初級 / 上級**）。
- 慣用色 127 色を対象。
- マンセル Value の完全一致による正誤判定。
- カードのフリップ演出と、裏面のグレースケール可視化。
- 「惜しい！」表示（明度差が近い不正解のみ）。
- クリア演出＋「もっと続ける」による再挑戦。

### やらないこと

- スコア・タイマー・試行回数制限・ランキング。
- プレイ状態や難易度選択の永続化。
- ヘッダーへの導線（今回は保留）。
- 他のゲームモード（本ゲームの派生・別ルール）。
- 中級難易度（実装予定なし）。

## ユーザーストーリー・主要ユースケース

- 学習者が、基準色と同じ明度のカードを探すことで、色相・彩度に惑わされず明度を見る目を鍛える。
- 来訪者が、点数のプレッシャーなく何度でも気軽に挑戦し、裏面の解説で「なぜ正解／不正解か」を学ぶ。
- 間違えても気軽にやり直せることを価値とし、正解／不正解の feedback 自体が学びと手応えになる。

## 機能要件

### 画面構成

- 大きく表示した **基準色カード 1 枚**（8 枚には含めない）。
- **候補カード 8 枚**（枚数固定）。
- **「残り X 枚」の進捗表示**（このラウンドで見つけるべき正解の残数）。
- **難易度切替（初級 / 上級）**。

### 1 ラウンドの流れ

1. 基準色と候補 8 枚が配られる。
2. プレイヤーが候補カードをクリックする。
3. **フリップ演出**でカードが裏返り、裏面に結果が表示される。
4. 正解カードを **すべて** 見つけたら **クリア演出** を表示。
5. 「**もっと続ける**」ボタンで、新しい基準色＋候補 8 枚を配り直す（エンドレス）。

### 正誤判定

- カードのマンセル Value が **基準色の Value と完全一致なら正解**、そうでなければ不正解。
- マンセル Value は 0.5 刻みに量子化済みのため、「完全一致」で明快に判定できる。

### 正解枚数

- **1〜3 枚（ランダム）**。「残り X 枚」でカウントダウン表示する。

### 不正解時の挙動

- カードを **表に戻して選び直す**、または **裏のまま残して他のカードを追加で探す**、どちらも可能。
- 不正解に対するペナルティ（減点・失敗）は無い。

### カードの裏面

- 「**正解 / 不正解**」テキストを表示。
- **マンセルのグレースケール軸**（明度スケール）上に、**基準色と選択色を明度の高さで並べて**表示し、両者の明度差を可視化する。
- **Value 差が 0.5 以内の不正解のみ「惜しい！」** テキストを追加表示する（0.5 を超える差は通常の「不正解」）。
- 色名（例: 桜色）やマンセル値など、学習を補助する情報を併記してよい。

### 難易度

- **初級**
  - はずれ札は基準色と **色相が近い色** で揃える。
  - **正解札は色相制約なし**（基準色と同 Value の色を全色相から拾う）。似た色が並ぶ中で、明度で正解を見抜く自然な難度になる。
- **上級**
  - 候補札の **色相・彩度をわざとバラバラ** に出す。色相の手がかりが使えず、純粋に明度で判断する必要がある。
- **共通**
  - はずれ札の Value は、全難易度共通で **基準色から ±0.5〜±1.0** の範囲に散らす（＝画面は常に「明度が近い 8 枚」になる）。
- 「色相が近い」は **マンセル色相距離** で測る。

## 既存機能との関係・整合

- 新ルート **`app/src/routes/games/lightness-match`** を新設する。ゲーム系ページは初めてで、新カテゴリの追加となる。
- **トップページに「色を見分ける」セクションを新設**し、本ゲームへのリンクカードを並べる。
- **ヘッダーへの導線は保留**（今回は実装しない）。
- 既存の `analyze`（配色分析）・`approximate`（近似色検索）とは独立した機能。色データ・明度計算まわりのユーティリティは共有する。
- データは既存の **`jis_colors.json`（全 127 色）** を利用。明度は **`app/src/lib/color/munsell.ts`** で Value をパースして取得。無彩色（グレー）の描画や明度計算は既存の `chroma-js` / `lightnessContrast.ts` の実績を流用できる。

## ドメインルール上の考慮

- 「明度」= **マンセル Value**。PCCS のトーンとは独立に扱う（同一トーンでも色相によって実際の明度は異なるため、"同一トーン = 同一明度" にはしない）。
- ヘルムホルツ–コールラウシュ効果により、Value が同じでも鮮やかな色ほど明るく "見える"。しかし **正解の根拠は見た目ではなく Value** に置く。この錯覚を見抜くこと自体が本ゲームの狙いである。

## データ・状態

- 127 慣用色（`name` / `munsell` / `_hex` など）を読み込み、マンセル Value を算出する。
- ラウンド状態として以下を保持する。
  - 基準色
  - 候補 8 枚（各色・表裏状態・正誤）
  - 残り正解数
  - 選択中の難易度
- **永続化しない**。難易度選択・プレイ状態は **ページアクセスのたびに都度リセット** する。

## エッジケース・異常系

- **同 Value 在庫が薄い基準色**：V=1.0 / 1.5 / 2.0（各 1 色）・V=9.0（2 色）など、同 Value の相手が足りず正解札（1〜3 枚）を作れない色は、**基準色プールから除外** する。
- **はずれ札に基準色と同 Value を混ぜない**（意図しない正解の発生を防ぐ）。
- **初級で近色相の在庫が不足する場合**：正解札はもともと色相制約が無いので確保できる。はずれ札は近色相を優先し、足りなければ色相条件を緩めてフォールバックする。
- **無彩色（グレー系）が基準色の場合**：マンセル色相が定義されないため、初級の「近い色相」ルールを適用できない。無彩色を初級の基準色から除外するか、近色相条件をスキップするかは実装時に決める（下記「未確定」参照）。
- **正解枚数と Value 在庫の整合**：ランダムに決めた正解枚数（1〜3）が、その基準色の同 Value 在庫を超えないよう、在庫に応じてクランプする。

## 非機能要件

- **モバイル**：基準色カード＋候補 8 枚のグリッド＋フリップ演出が、スマートフォンの画面幅でも成立するレイアウトにする。
- **アクセシビリティ**：本質的に視覚（明度知覚）に依存するゲームである。ただし明度弁別は色覚特性の影響を受けにくく、色覚多様性には比較的頑健。カード選択はキーボード操作でも行えるようにする。
- **パフォーマンス**：対象 127 色・候補 8 枚と軽量で、生成・描画の負荷懸念は無い。

## 制約

- Svelte 5 / SvelteKit、`chroma-js` を用いる。既存のデータ構造・ユーティリティに準拠する。
- `.svelte` 内で Map を使う場合は `SvelteMap` を用いる（Record への置換はしない）。
- コード実装後は `app` ディレクトリで `npm run check` を通す。

## 受け入れ条件

- 初級・上級それぞれで、基準色＋候補 8 枚が正しく生成される（正解 1〜3 枚、はずれ札は基準色から ±0.5〜±1.0 の近明度、同 Value の混入なし）。
- 候補カードをクリックするとフリップ演出で裏返り、Value 一致で正解、不一致で不正解と判定される。
- カード裏面に、グレースケール軸上へ基準色と選択色を明度の高さで並べた可視化が表示され、Value 差 0.5 以内の不正解には「惜しい！」が表示される。
- 正解カードをすべて見つけると、クリア演出が表示され、「もっと続ける」で新しい基準色＋候補 8 枚が配り直される。
- トップページに「色を見分ける」セクションが追加され、本ゲームページへ遷移できる。
- 難易度・プレイ状態がページアクセスのたびにリセットされる。
- `npm run check` がエラー・警告なく通る。

## 未確定・保留事項

- **無彩色（グレー系）を基準色にした場合の初級の扱い**（近色相ルールをスキップするか、初級の基準色プールから除外するか）。
- **ヘッダーへの導線**（今回は保留。将来的に追加する可能性）。
- **クリア演出・フリップ演出の具体的な表現**（アニメーションの内容）。
- 「色相が近い」を測るマンセル色相距離の **具体的なしきい値**（何色相以内を「近い」とみなすか）。
