# グレースケール・ヒント（明度マッチング）

明度マッチングゲーム（`/games/lightness-match`）に、カードの下半分をグレースケールで表示する **ヒント機能** を追加する。ON にすると全カードの下半分が「その色のマンセル Value から作ったグレー」で塗られ、実際の色とその明度を 1 枚の中で見比べられるようになる。

## 目的・背景

- 明度マッチングの **難易度を下げる「ヒント」** を提供する。詰まったプレイヤーが、明度そのものを直接見て正解を探せるようにする。
- 同時に、**実際の色とそのグレー（明度）を並べて見せる**ことで、プレイヤーが「脳内でグレースケール化して明度を見る目」を養えるようにする。難易度緩和と学習支援の両面を狙う。

## スコープ

### やること

- ヒントの **ON/OFF トグル**（**既定 OFF**）を 1 つ追加する。ON で **全カード一斉** に効く。
- ON 時、各カード（**候補 8 枚＋基準色カード**）の **下半分（上下 50 / 50・境界線なし）** を、その色の **マンセル Value から作ったグレー**（`grayHexForLightness(value * 10)`）で塗る。上半分は実際の色のまま。
- ON/OFF 状態は **セッション中は保持** する（「もっと続ける」で次ラウンドに進んでも、モードを切り替えても維持）。
- ON/OFF 状態は **ページ再読み込みで OFF にリセット** する（永続化しない）。
- **両モード（色相が近い／彩度が近い）共通** で有効。

### やらないこと

- ON/OFF 状態の **localStorage 等への永続化**。
- **CSS `filter: grayscale()` 方式**（luma 基準になり、Value 判定と食い違って誤誘導しうるため不採用）。
- **クリア演出のスウォッチ**（`ClearSwatches`）へのグレー化適用（フルカラーのまま）。
- 難易度（モード）によるヒント挙動の差別化。
- グレー半分そのものに対する個別の ARIA 読み上げ。

## ユーザーストーリー・主要ユースケース

- 学習者が正解を見つけられず詰まったとき、ヒントを ON にして下半分のグレーを手がかりに、明度が一致するカードを探す。
- 学習者が、色つきの上半分を見て頭の中でグレー化を試み、ヒントを ON にして答え合わせすることで、明度を見る感覚を較正する。
- カジュアルな来訪者が、難しいと感じたらヒントで気軽に難易度を下げて遊ぶ。

## 機能要件

### トグルボタン

- **素の HTML でアクセシブル** になるマークアップを優先する（ネイティブの `<input type="checkbox">` ＋ `<label>` のように、状態が HTML だけで露出する形）。`aria-pressed` などの ARIA 属性には頼らない。
- キーボード操作・状態把握がネイティブに成立すること。
- ラベルは「**ヒント**」＋下半分グレー化の意味を添える短い補足。
- 難易度切替（`.controls`）と同じ並びに置く場合は **右寄せ**。トグル追加後もコントロール行がスマートフォン幅で破綻しない **レスポンシブ** を考慮する。
- 最終的な配置・文言は後で調整する（未確定）。

### カード表面のグレー帯

- ヒント ON のとき、カード表面（`GameCard` の `front`）の **下半分** に、その色の Value 由来グレーの帯を敷く。
  - グレー色 = `grayHexForLightness(value * 10)`（裏面の明度軸 `LightnessAxis` と同一の換算）。
  - 上下 **50 / 50**、境界は **ハードエッジ（区切り線なし）**。上半分は実際の色。
- ヒント OFF のときは従来どおり色べた塗り。
- 基準色カードにも同じ上下分割を適用する。

### 判定との関係

- グレーが Value 由来のため、**同じ Value のカードは基準色と完全に同一のグレー** になる（正誤判定＝Value 完全一致と 1:1 で一致し、ヒントが誤誘導にならない）。
- 「惜しい！」相当（Value 差 0.5）のカードは L\* で約 5 の差＝わずかに違うグレーとして残り、見分けの余地は残る。

## 既存機能との関係・整合

- `GameCard`（`app/src/lib/components/games/GameCard.svelte`）の表面に、ヒント時の下半分グレー帯を **オプトイン** で追加する。同コンポーネントは tone-match / tone-hunt でも使われるため、プロパティ未指定時は従来挙動を保ち、既存ゲームを壊さないこと。
- 基準色カードは現状 `+page.svelte` 内の素の `<div class="base-card">`。同じ上下分割に対応できるよう **リファクタ** する。
- グレー生成は既存の `grayHexForLightness`（`app/src/lib/color/lightnessContrast.ts`）を流用する。
- グレーの元になる `value` / `baseValue` は、既にラウンドデータ（`CandidateColor.value` / `Round.baseValue`）に存在する。追加のデータ取得は不要。

## ドメインルール上の考慮

- グレーは **マンセル Value → グレー**（裏面の明度軸と同一換算）で作る。CSS の luma や CIELAB L\* の直接脱色ではなく、**Value 判定と矛盾しない** ことを最優先する。
- これにより「同じ Value ＝ 同じグレー」を体験でき、ゲームの土俵（マンセル Value）の上での“正しい脱色”を学習支援として提示できる。

## データ・状態

- `hint`（真偽値）をページのセッション状態として保持する（既定 `false`）。
- 永続化しない。ページアクセス（再読み込み）のたびに `false` に戻る。
- ラウンド再生成・モード切替では `hint` の値を保持する（リセットしない）。

## エッジケース・異常系

- **無彩色の候補札**：`grayHexForLightness(value * 10)` のグレーがほぼ元の色と一致するが、無害（誤誘導は起きない）。
- **フリップ済みカード**：表面が隠れるためヒントの影響を受けない。表に戻せばグレー帯が再表示される。
- **クリアオーバーレイのスウォッチ**：ヒント対象外。フルカラーのまま表示する。

## 非機能要件

- **アクセシビリティ**：トグルは ARIA 属性に頼らず、ネイティブ HTML で状態が露出する形にする。グレー帯は視覚的補助であり、個別の読み上げは付けない。
- **モバイル**：トグル追加後もコントロール行が成立するレスポンシブにする。
- **パフォーマンス**：追加計算はカード 1 枚あたり Value からのグレー算出 1 回程度で、負荷懸念はない。

## 制約

- Svelte 5 / SvelteKit、`chroma-js` を用いる。既存のデータ構造・ユーティリティに準拠する。
- トグルは `aria-pressed` 等ではなく、素の HTML でアクセシブルにする（本機能特有の方針）。
- `.svelte` 内で Map を使う場合は `SvelteMap` を用いる。
- コード実装後は `app` ディレクトリで `npm run check` を通す。

## 受け入れ条件

- 既定 OFF で、従来どおりゲームが遊べる。
- トグル ON で、基準色カード＋候補 8 枚すべての下半分が Value 由来グレー（上下 50 / 50・境界線なし）になり、上半分は実際の色のまま表示される。
- ON のとき、同じ Value のカードの下半分が基準色の下半分と完全に同一のグレーになる。
- ヒント ON/OFF が「もっと続ける」・モード切替をまたいで保持され、ページ再読み込みで OFF に戻る。
- トグルが ARIA 属性なしで、キーボード操作・状態把握ともにネイティブに成立する。
- tone-match / tone-hunt など `GameCard` を共有する既存ゲームの表示・挙動が変わらない。
- `npm run check` がエラー・警告なく通る。
