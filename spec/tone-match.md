# PCCSのトーンを見分ける（tone-match）

トーンマップで選んだ 1 つのトーンを提示し、そのトーンに属する色を候補カードの中から探す選択式のカードゲーム。清濁 3 グループを見分ける `tone-hunt` より一段細かく、**個々のトーンの同定**を鍛える。

## 目的・背景

- 色彩検定などを見据え、「**色を見てそのトーンを識別できる**」力を鍛えることが狙い。分類（明清色 / 暗清色 / 中間色）を見抜くだけでは不十分で、色から個々のトーンを答えられる必要がある。
- 既存の `tone-hunt`（清色・濁色を清濁 3 グループで見分ける）を土台に、その先の **個々のトーン単位の弁別** を扱う。
- 想定プレイヤーは、**トーンを一通り学んだ人**（初学者向けではなく、一段踏み込んだ層）。
- 選択式のため厳密な「名指し」ではなく「**そのトーンに属する色を見分ける**」訓練になるが、お題選択をトーンマップ上のクリックで行い、隣接トーンを絡めることで、実質的なトーン弁別力を鍛えられる。

## スコープ

### やること

- 選択式のカードゲーム 1 種（**単一モード**）。
- トーンマップから探すトーンを選ぶ UI。
- 有彩色 11 トーンを対象にした出題。
- お題トーンへの所属による正誤判定。
- カードのフリップ演出と、裏面のトーン表示（実トーン記号＋和名）。
- 「惜しい！」表示（お題トーンの隣接トーンを選んだ不正解のみ）。
- クリア演出（`ClearOverlay`）＋「もっと続ける」による次トーンの出題（セッション進行）。

### やらないこと

- 難易度段階（初級 / 上級など）。**単一難易度**。
- 「色 → トーン名を当てる」逆方向モード（検討したが廃止）。
- スコア・タイマー・試行回数制限・ランキング。
- プレイ状態やクリア進捗の永続化。
- **キーボード操作対応**（全ゲーム横断で別途統一対応する予定のため、今回はスコープ外）。
- `s` トーン・奇数色相の出題。
- 無彩色トーン（W / Gy / Bk）を「お題（探す対象）」にすること。
- 全トーンクリア時の演出（無演出でリセット）。

## ユーザーストーリー・主要ユースケース

- 学習者が、トーンマップで選んだトーン（例: `lt`）に属する色を候補から探すことで、そのトーンの色の見え方を反復して身につける。
- 隣接トーン（例: `lt` と `b` と `p`、`d` と `sf`）が候補に混ざることで、紛らわしいトーン同士の弁別を迫られ、試験で問われる識別力が鍛えられる。
- 間違えても気軽にやり直せることを価値とし、裏面のトーン表示と「惜しい！」が学びと手応えになる。
- 全 11 トーンを 1 つずつ潰していくことで、ひととおりのトーンを網羅的に練習できる。

## 機能要件

### 画面構成

- **トーンマップ**（探すトーンを選ぶ UI。全トーンが常時選択可能）。
- 現在のお題を示す **プロンプト**（例: 「`lt` トーンを探そう」）。
- **「残り X 枚」の進捗表示**（このラウンドで見つけるべき正解の残数）。
- **候補カード 8 枚**（枚数固定）。

### 1 ラウンドの流れ

1. お題トーンと候補 8 枚が提示される（初期状態は **v トーン**が選択済み）。
2. プレイヤーが候補カードをクリックする。
3. **フリップ演出**でカードが裏返り、裏面に結果が表示される。
4. 正解カードを **すべて** 見つけたら **クリア演出**（`ClearOverlay`）を表示。
5. 「**もっと続ける**」で **未クリアのトーンからランダムに** 次のお題を出題する（クリア済みトーンは自動選出しない）。

### トーン選択と進行

- **初期状態**：`v` トーンが選択済みで盤面が表示される。
- **手動選択**：トーンマップは **全トーンが常時選択可能**（クリア済みかどうかを区別・ロックしない）。プレイヤーはいつでも任意のトーンを選び直せる。
- **自動進行**：クリア後「もっと続ける」を押すと、**未クリアトーンからランダム**に次のお題を選ぶ。
- **クリア済み管理**：セッション内で「クリア済みトーン集合」を保持する（自動出題の対象外にするためだけに使う。マップ表示には反映しない）。
- **全トーンクリア**：11 トーンすべてをクリアしたら、**リセットして最初から**（初期状態＝`v` に戻す）。演出は挟まない。
- 手動選択・進行状態は **永続化しない**。ページアクセスのたびに都度リセットする。

### 正誤判定

- カードのトーンが **お題トーンと一致すれば正解**、そうでなければ不正解。

### 正解枚数

- **1〜3 枚（ランダム）**。「残り X 枚」でカウントダウン表示する。

### 不正解時の挙動

- カードを **表に戻して選び直す**、または **裏のまま残して他のカードを追加で探す**、どちらも可能。
- 不正解に対するペナルティ（減点・失敗）は無い。

### はずれ札の選定

- **なるべく隣接トーンから** 選ぶ。「隣接」は **トーンマップ上の縦・横・斜め**（`pccs-tone.ts` の隣接判定）で定義する。
- 隣接集合に `s` が含まれる場合は、`s` がプール外のため除外する。
- **たまに無彩色を 1 枚** 混ぜる枠を確保する（変化球）。その無彩色は **お題トーンの明度水準に近いグレー**（高明度トーン → W / 明るい灰、中明度トーン → 中灰、低明度トーン → 暗い灰 / Bk のように対応付け）を選ぶ。
- 正解トーンの混入・同一色の重複は起こさない。

### カードの裏面

- 「**実トーン記号＋和名**」（例: `sf` ソフト）と **正誤** を表示する（それ以外の情報は載せない。トーンマップ上の位置表示などは行わない）。
- 選んだ不正解が **お題トーンの隣接トーン（縦横斜め）** だった場合のみ「**惜しい！**」を追加表示する。

## 既存機能との関係・整合

- 新ルート **`app/src/routes/games/tone-match`** を新設する。ゲーム系（`games/`）配下の 3 つ目のゲーム。
- **トップページ「色を見分ける」セクション** にリンクカードを追加する。
- 既存 `games/tone-hunt`（清濁 3 グループの分類）とはお題の粒度が異なる別ゲーム。名称・スラッグを分けて混同を避ける。
- `ClearOverlay`（`app/src/lib/components/games/`）を流用する。
- トーンデータ・隣接判定は既存の **`app/src/lib/data/pccs-tone.ts`**（縦横斜めの隣接関係）と **`app/src/lib/data/pccs.ts`**（`PCCS_CARD_199` など）を利用する。
- 無彩色の明度近接判定は、`lightness-match` で用いているマンセル Value 取得（`app/src/lib/color/munsell.ts`）の実績を流用できる。

## ドメインルール上の考慮

- 対象は有彩色 **11 トーン**（v / b / dp / lt / sf / d / dk / p / ltg / g / dkg）。`s` トーンと奇数色相は「新配色カード199」準拠プールに含まれないため対象外。
- 「隣接トーン」= トーンマップ上で縦・横・斜めに隣り合うトーン（`pccs-tone.ts` の定義に準拠）。既存の隣接判定は `s` を含むため、プール外の `s` は除外して扱う。
- `v` トーンはトーンマップの縦・横グループに乏しいが、斜め隣接（b / sf / d / dp）が取れるため、隣接はずれ札を確保できる。
- 無彩色（W / Gy / Bk）は色相を持たず、お題（探す対象）にはしない。distractor としてのみ、お題トーンの明度水準に近いグレーを混ぜる。

## データ・状態

- 有彩色プール：`PCCS_CARD_199` の有彩色（`v` 全 24 色相 ＋ その他偶数色相 120 色 ＝ 144 色。`s`・無彩色・奇数色相を含まない）。
- 無彩色プール：distractor 用のグレー段階（W / 明灰 / 中灰 / 暗灰 / Bk など）。
- ラウンド状態として以下を保持する。
  - お題トーン
  - 候補 8 枚（各色・表裏状態・正誤）
  - 正解枚数（1〜3）と残り正解数
  - クリア済みトーン集合（セッション内・自動出題の対象制御にのみ使用）
- **永続化しない**。ページアクセスのたびに都度リセットする。
- `.svelte` 内で Map を使う場合は `SvelteMap` を用いる。

## エッジケース・異常系

- **正解枚数と在庫の整合**：各トーンは 12 色相以上（`v` は 24）の在庫があり、正解 1〜3 枚は常に確保できるが、念のため在庫でクランプする。
- **隣接トーン在庫の不足**：隣接トーンを優先して埋め、それでも 8 枚に満たない場合は非隣接の有彩色トーンでフォールバックする。
- **`s` の除外**：隣接集合に `s` が含まれてもプールに存在しないため候補にはならない。
- **無彩色 distractor**：混ぜる際はお題トーンの明度水準に近いグレーを選ぶ。混ぜない盤面もありうる（毎回は入れない）。
- **全トーンクリア**：クリア済みが 11 トーンに達したら、クリア済み集合をリセットし初期状態（`v`）に戻す。
- **クリア済みトーンの手動選択**：マップからはクリア済みトーンも選べる（クリア判定・集合には既に含まれるため進行に影響しない）。

## 非機能要件

- **アクセシビリティ**：本ゲームは本質的に **色相・彩度・トーンの弁別に依存** し、明度弁別中心の `lightness-match` と違って **色覚タイプの影響を受ける**。これは既知の制約として明記する。**キーボード操作対応は今回スコープ外**（全ゲーム横断で別途統一対応する）。
- **モバイル**：トーンマップ＋候補 8 枚のグリッド＋フリップ演出が、スマートフォンの画面幅でも成立するレイアウトにする（`tone-hunt` 同様、狭幅で 2 列へ折り返すなど）。
- **パフォーマンス**：対象 144 色・候補 8 枚と軽量で、生成・描画の負荷懸念は無い。

## 制約

- Svelte 5 / SvelteKit。既存のデータ構造・ユーティリティに準拠する。
- `.svelte` 内で Map を使う場合は `SvelteMap` を用いる（Record への置換はしない）。
- 既存ゲーム（`lightness-match` / `tone-hunt`）のディレクトリ構成・命名に揃える（`app/src/routes/games/tone-match/`、`app/src/lib/games/tone-match/round.ts` ＋ `round.spec.ts`、`app/src/lib/components/tone-match/`）。
- `+page.ts` は `export const ssr = false`。
- コード実装後は `app` ディレクトリで `npm run check` を通す。

## 受け入れ条件

- トーンマップから任意のトーンを選ぶと、そのトーンがお題になり、候補 8 枚（正解 1〜3 枚、はずれ札はなるべく隣接トーン、正解トーンの混入なし）が生成される。
- ページを開いた直後は `v` トーンが選択済みで盤面が表示される。
- 候補カードをクリックするとフリップ演出で裏返り、お題トーン一致で正解、不一致で不正解と判定される。
- カード裏面に「実トーン記号＋和名」と正誤が表示され、お題トーンの隣接トーン（縦横斜め）を選んだ不正解には「惜しい！」が表示される。
- はずれ札にはなるべく隣接トーンが含まれ、盤面によってはお題トーンと明度の近い無彩色が 1 枚混ざる。
- 正解カードをすべて見つけるとクリア演出が表示され、「もっと続ける」で未クリアトーンからランダムに次のお題が出題される（クリア済みは自動選出されない）。
- 11 トーンすべてをクリアすると、演出なしでリセットされ初期状態（`v`）に戻る。
- トップページ「色を見分ける」セクションに本ゲームへのリンクカードが追加され、遷移できる。
- プレイ状態・クリア進捗がページアクセスのたびにリセットされる。
- `npm run check` がエラー・警告なく通る。

## 未確定・保留事項

- 無彩色 distractor を混ぜる **頻度**（何ラウンドに 1 回程度か。実装で調整）。
- 「明度が近い無彩色」の **具体的な対応付け**（トーンの明度水準 ↔ グレー段階のマッピング）。
- ページ見出しの **アイコン**（`tone-hunt` は `mdi:invert-colors` を使用）。
