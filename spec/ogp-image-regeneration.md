# OGP画像の記録と一括再生成

## 目的・背景

- OGP 画像生成の仕組み（`spec/ogp-image-generation.md`）はあるが、**リデザイン時などに全 OGP 画像を手軽に再生成する手段がない**。
- スキル（`generate-ogp-image`）が対話で解決した値（タイトルの改行位置・パンくず crumb・図版）が**永続化されていない**。特に図版は都度手渡しの一時パスで、生成後は消えてしまう。
- そのため、テンプレートをリデザインすると、全ページ分の対話（改行提案・図版手渡し）を**やり直す**ことになる。
- スキルの解決結果を**記録として永続化**し、記録から**対話ゼロで一括再生成**できる仕組みを追加する。

## スコープ

### やること

- **記録データ層** `ogimage/data/`（ルートごと 1 ファイルの JSON）。
- **図版アセット層** `ogimage/data/assets/<route>/`（手渡し図版の永続コピー）。
- **`ogimage/render.mjs` の拡張** … 記録ファイル・図版コピー・マニフェスト更新を担う唯一の書き手にする。
- **一括再生成スクリプト** `ogimage/regenerate.mjs`（`npm run regenerate`）。
- **`og-manifest.json` の派生物化**（案B：`ogimage/data` を正、マニフェストはそこから導出）。
- **スキル `generate-ogp-image` の単体再実行 UX** … 既存記録があれば図版・改行を再利用する。

### やらないこと（保留／運用境界）

- 記録の title・カテゴリと現コードの**ドリフトを警告する audit**（コード再走査が要る＝知能層のため保留）。
- 削除ページの**自動 prune**（手動で後始末する）。
- **コンテンツ変更（title・カテゴリ）に追従した記録の自動更新**（＝運用境界。該当ページをスキルで個別再生成する）。
- タイトルの**自動折り返し**（手で決めた改行を保存・再利用する方針のため）。
- 複数図版対応（現状 1 ページ 1 枚前提）。

## ユーザーストーリー・主要ユースケース

- テンプレートをリデザインした著者が、`npm run regenerate` で**記録のある全ページ＋既定画像**を対話ゼロで一括再生成し、崩れたページだけスキルで個別に直す。
- リデザインが特定区分だけに効いたとき、`node ogimage/regenerate.mjs '/color-theory/*'` のように **glob で部分再生成**する。
- 記事の title やカテゴリを変更した著者が、そのページを**スキルで個別再生成**する（記録も更新される）。
- 図版入りページを再生成する際、**前回手渡した図版が `data/assets` に残っている**ので、手渡しをやり直さずに済む。

## 機能要件

### 責務分離（既存方針を継承）

- **スキル（知能層）**：コードを読んでバリエーション・タイトル・crumb を解決し、改行や図版の要否を判断し、確定値を JSON で render.mjs に手渡す。
- **描画スクリプト（描画＋簿記）**：受け取った確定値を PNG 化する。加えて、確定値の**記録**・図版の**永続コピー**・マニフェストの**更新**という簿記を担う（マニフェスト更新は既存の副作用の延長）。
- **一括再生成スクリプト（描画専用の自動化）**：記録を読んで render.mjs の一括経路に流すだけ。コード走査・メタ抽出・インタビューは行わない。

### 記録の書き込み（render.mjs）

- `variation` が `default` **以外**の全生成で、per-route 記録 `ogimage/data/<route>.json` を書く（**title-only ページも書く**。書かないと一括再生成のスイープから漏れるため）。
- 図版（`figure`）が渡された場合、`ogimage/data/assets/<route>/figure.<拡張子>` へ**コピー**し、記録の `figure` をこの**永続パス**（`data` 基準の相対）に書き換える。
  - ファイル名は**固定ベース名 `figure`＋元の拡張子**。同一パスへ上書き（冪等）で、差し替え時にゴミが残らない。
- マニフェスト（`app/src/lib/meta/og-manifest.json`）は、単発・部分再生成時は従来どおり **upsert**。
- `default`（トップ）は記録もマニフェストも書かない（従来どおり）。

### 記録ファイルのスキーマ

```jsonc
// ogimage/data/color-theory/pccs-basics.json
{
  "route": "color-theory/pccs-basics",       // 先頭/末尾スラッシュ無し
  "title": "PCCSと色の分類",                   // og:title 用の完全なタイトル
  "titleLines": ["PCCSと", "色の分類"],        // 描画用の改行済みタイトル（1〜2 要素）
  "crumbs": ["色の理論", "PCCSと色彩調和"],     // nested 系のみ。title-only では空/省略
  "figure": "assets/color-theory/pccs-basics/figure.png" // 図版があるときのみ（data 基準の相対パス）
}
```

- **`variation` は持たない**。再生成時に route から `config.mjs` の `resolveVariation(route, { hasFigure })` で引き直す（案ii）。
  - バリエーション規則（`OG_RULES`）の変更は `config.mjs` 一箇所で全再生成に効く、という一貫性を保つため。
  - 図版の有無による `optional → nested-fig` の昇格だけは、記録の `figure` の有無で決まる。
- `figure` は `ogimage/data/` を基準とした相対パス。render.mjs / regenerate.mjs はこれを `data` 基準で解決する。

### 一括再生成（`ogimage/regenerate.mjs` / `npm run regenerate`）

- **スキルを介さないプレーンなスクリプト**（対話ゼロ＝知能不要）。リポジトリのルートから `node ogimage/regenerate.mjs` で実行。
- 動作:
  1. `ogimage/data/**/*.json` を glob（`assets/` 配下は PNG なので JSON にヒットせず自然に除外）。
  2. 各記録を読み、route から `config.mjs` で `variation` を再解決（`figure` 有無で `nested-fig` 昇格）。
  3. render.mjs の**既存の一括経路**（配列 / `{ items: [...] }`）に流す。
  4. `default.png` を**無条件生成**する。
- **記録があるページ＝再生成対象**。記録の存在そのものがレジストリで、別インデックスは持たない。新規/未生成ページは対象外（スキルでの初回生成が必要）。
- **部分再生成**：オプションの glob 引数で対象を絞れる（例: `node ogimage/regenerate.mjs '/color-theory/*'`）。
- **マニフェストの更新方式**:
  - **全再生成時**：記録の全集合から**マニフェストを rebuild**（作り直す）。削除ページの項目も自動で消え、派生物として自己修復する。
  - **部分再生成・スキルの単体生成時**：**upsert**（対象外ページの項目を消さない）。
- **頑健モード**：per-page でエラーを握って**続行**し、最後に「成功 N 件／失敗 M 件＋失敗一覧」を報告する。1 ページの不備で全体を止めない。
  - 単発の render.mjs 直叩き（スキル経由）は従来どおり **fail-fast**（全件検証してから描画）。

### スキル単体再実行時の記録再利用

- スキルは再実行時、対象ルートに**既存の記録があればそれを既定値として再利用**する（自分が書いた記録を読むだけで、アプリのコード走査ではない＝責務分離は崩れない）。
  - **図版**：`data/assets` の保存済みアセットを既定で流用する（差し替えたいときだけ新規手渡しを受け取る）。
  - **改行**：保存済み `titleLines` を改行案の**初期提案**として提示する（修正可）。

## 既存機能との関係・整合

- **`og-manifest.json`**：案B により**派生物**になる。`<SiteMeta>` の参照内容（route の存在＋ `title`）は不変。
- **`config.mjs`**：`variation` の正であり続ける（案ii）。`resolveVariation` を regenerate.mjs が使用する。
- **`render.mjs`**：スキルからの**入力インターフェースは不変**（従来の JSON をそのまま渡す）。副作用として記録書き込み・図版コピーが増える。
- **`generate-ogp-image` スキル**：単体再実行時の記録再利用ロジックを追加し、SKILL.md を更新する。
- 既存の `spec/ogp-image-generation.md` を置き換えるのではなく、その上に**記録・再生成の層を追加**する位置づけ。

## ドメインルール上の考慮

- 本機能は色分析ロジック（PCCS の色相・トーン、色距離、配色技法等）に依存しない。ドメインルールとの矛盾は生じない。

## データ・状態

| パス | 役割 | 追跡 |
| --- | --- | --- |
| `ogimage/data/<route>.json` | 記録（**正**） | コミット |
| `ogimage/data/assets/<route>/figure.<ext>` | 図版アセット（**正**） | コミット |
| `app/src/lib/meta/og-manifest.json` | 派生（記録から導出） | コミット |
| `app/static/ogp/<route>.png` | 生成物（出力画像） | コミット |

- `.gitignore` は `node_modules` / `.DS_Store` のみ。`ogimage/data/` は追加設定なしでコミット対象になる（変更不要）。
- 定型プレビュー（`/jis-color-map/<family>`, `/patterns/<theme>`）の手渡し PNG も、同じ仕組みで記録＆ `data/assets` に配置し、一括再生成の対象に含める（当面手渡しである点は不変）。

## エッジケース・異常系

- **図版アセット欠損／記録の route が `config.mjs` の規則に非該当**：一括再生成では **per-page 失敗として続行・報告**。単発 render.mjs では fail-fast。
- **ページ削除**：記録・アセット・出力 PNG を**手動削除**する。全再生成でマニフェストは自己修復（削除ページの項目が消える）。
- **コンテンツ変更後に一括再生成**：記録が古いままだと**古いタイトルで生成されうる**（運用境界。該当ページはスキルで個別再生成する）。
- **リデザインで改行が収まらない**：render.mjs の font-size 自動縮小が効く。破綻・意図ズレは**目視**で見つけ、崩れたページのみスキルで個別再生成。
- **記録の再利用（スキル単体）で保存済み図版が欠損**：新規手渡しを促す。

## 非機能要件

- **見た目確認**：出力 PNG の描画（日本語・英字ロゴ・crumb・図版）は**ユーザーが目視確認**する（自動視覚検証・ヘッドレスブラウザは使わない。既存方針）。
- **パフォーマンス**：手動・オンデマンド。記録の glob＋パースはミリ秒オーダーで問題にならない。
- **アクセシビリティ**：`og:image:alt`（タイトル由来）は既存のまま。新たな a11y 懸念なし。
- **保守性**：バリエーション規則は `config.mjs` に集約したまま。記録は per-route で編集・削除・マージ衝突に強い。

## 制約

- 既存の OGP 生成の制約を継承する:
  - resvg（usvg）は `var()` / `light-dark()` を解決できない。図版は自己完結（手渡し PNG）。
  - ヘッドレスブラウザ（Playwright）でのスクショ取得は行わない。
  - アプリは `@sveltejs/adapter-static`、`trailingSlash: "always"`。
- `ogimage/` は app の Svelte/Vite 実行環境に結合しない（独立性を維持）。regenerate.mjs も同様。

## 受け入れ条件

- スキルで単体生成すると、**記録 JSON が書かれ**、図版が `data/assets/<route>/figure.<ext>` に配置され、記録の `figure` が永続パスに書き換わる（title-only では crumb/figure 無しの記録が書かれる）。
- `npm run regenerate` で、**記録のある全ページ＋ `default.png`** が再生成され、マニフェストが記録全集合から **rebuild** される。
- glob 引数で**部分再生成**でき、そのときマニフェストは **upsert**（対象外ページの項目を消さない）。
- 一括再生成中、1 ページの不備で**全体が止まらず**、最後に成功/失敗が一覧される。
- スキル単体再実行時、**既存図版が流用**され、保存済み改行が**初期改行案**として提示される。
- 再生成後の PNG が意図どおり描画される（目視）。
