# CSS相対色構文の活用例

CSS相対色構文（CSS Relative Color Syntax）は、既存の色を基準として、その色の特定チャンネルだけを変化させた新しい色を生成できる記法です。
`oklch(from <元の色> <L> <C> <H>)` のように、元の色の各チャンネル値を `l`・`c`・`h` などの変数として参照しながら計算式で変換できます。

このドキュメントでは、pccs-lens アプリの Svelte コンポーネント内で CSS 相対色構文が実際にどのように活用されているかを、使用箇所ごとに解説します。

## 1. 色相環のハイライト枠線（`HueWheel.svelte`）

### 使用箇所

PCCS 色相環の SVG コンポーネントで、配色パターンに含まれる色相（ハイライトされた扇形）に強調枠線を描画する箇所。

```svelte
stroke={`oklch(from ${HUE_COLORS[h]} calc(l * .85) c h)`}
```

### 色の操作

- **元の色**: 色相 `h` に対応する色相環の扇形の塗りつぶし色（`HUE_COLORS[h]`）
- **操作**: OKLCHの明度（`l`）を元の値の85%に下げる。彩度（`c`）と色相（`h`）はそのまま維持する。

```
oklch(from <塗り色> calc(l * .85) c h)
           ↑ 元の色の l, c, h を参照できる
```

### 実現している効果

塗りつぶし色と同じ色相・彩度を持ちながら、わずかに暗い色を枠線に設定することで、セルに立体感を与えてハイライト状態を強調している。枠線色を塗り色と全く別の色にせず、あくまで同系色に留めることで、色相環全体の色調の統一感が保たれる。

## 2. トーン図セルの枠線（`ToneDiagram.svelte`）

### 使用箇所

PCCS トーン図の SVG コンポーネントで、ユーザーが登録した色が塗られているセルの枠線色を決定する箇所。

```svelte
{@const strokeColor = isUsed
  ? `oklch(from ${cell.usedColors[0].hex} calc(l * .85) c h)`
  : "#ccc"}
```

### 色の操作

- **元の色**: そのセルに紐づく登録済み色のHEX値
- **操作**: OKLCHの明度を85%に下げ、彩度・色相は維持する。

### 実現している効果

セルの塗り色より少し暗い同系色を枠線に使うことで、どんな色が塗られていても自然に見える枠線が得られる。色が設定されていない場合はニュートラルな `#ccc` を使うため、未使用セルとの視覚的な区別もできる。

## 3. トーン図セルのラベル文字色（`ToneDiagram.svelte`）

### 使用箇所

同じトーン図コンポーネントで、各セル上に表示されるラベルテキスト（トーン名など）の色を、セルの背景色に基づいて決定する箇所。

```svelte
{@const labelFill = isUsed
  ? isLightColor(cell.usedColors[0].hex)
    ? `oklch(from ${cell.usedColors[0].hex} calc(l - .60) c h)`
    : `oklch(from ${cell.usedColors[0].hex} calc(l + .60) c h)`
  : ...}
```

### 色の操作

- **元の色**: セルに紐づく登録済み色のHEX値
- **明るい色の場合**: 明度を `-0.60`（OKLCH は 0〜1 スケール）下げて、暗いテキスト色を生成する
- **暗い色の場合**: 明度を `+0.60` 上げて、明るいテキスト色を生成する
- いずれも彩度・色相はそのまま維持する

### 実現している効果

背景が明るい色であればテキストの明度を大きく下げ、背景が暗い色であればテキストの明度を大きく上げることで、どんな色のセル上でもテキストが読めるコントラストを確保している。明度を ±0.60 と大幅に変化させることで、隣接する明度差が十分確保される。彩度・色相を元の色のものを維持するため、テキスト色はセルの色に調和しながらもコントラストのある色になる。

## 4. 色相セレクターの選択中扇形の枠線（`HueSelector.svelte`）

### 使用箇所

色相選択 UI の SVG コンポーネントで、現在選択されている色相の扇形に強調枠線を描画する箇所。選択状態の枠線は「白い外枠」と「色相に合った内枠」の2重構成になっている。

```svelte
<!-- 選択中扇形の強調枠（外枠: 白） -->
<path fill="none" stroke="white" stroke-width="3" ... />
<!-- 選択中扇形の強調枠（内枠: 相対色） -->
<path
  fill="none"
  stroke={`oklch(from ${HUE_COLORS[h]} calc(l * .75) c h)`}
  stroke-width="1.5"
  ...
/>
```

### 色の操作

- **元の色**: 選択されている色相に対応する色相環の扇形の塗り色
- **操作**: OKLCHの明度を75%に下げる（HueWheel の85%よりさらに強め）。彩度・色相は維持する。

### 実現している効果

太い白枠と、塗り色より暗い色相内枠の2層構造により、選択状態が白背景・有彩色背景のどちらに対しても明確に認識できる。内枠の色が塗り色と同じ色相を持つため、「今この色相が選ばれている」という情報を視覚的に伝えながらも、色が浮いて見えることがない。

## 5. コピーボタンの枠線・ホバー背景・ホバー文字色（`CopyButton.svelte`）

### 使用箇所

HEX値などをクリップボードにコピーする汎用コンポーネントで、外部から `color` prop でベース色を受け取り、そのカラーを使って枠線・背景・テキスト色を計算する箇所。

```css
/* 通常状態の枠線 */
border: 1px solid rgb(from var(--color-copybtn-base) r g b / 35%);

/* ホバー時の背景 */
background: rgb(from var(--color-copybtn-base) r g b / 20%);

/* ホバー時のテキスト色 */
color: oklch(from currentColor calc(l * 0.75) c h);
```

### 色の操作

**枠線・ホバー背景**（`rgb(from ...)` 構文）

- `rgb(from <元の色> r g b / <alpha>)` という記法で、元の色のRGB値をそのまま取り出しつつアルファ値だけを変更する
- 枠線: アルファ値を35%に設定（うっすら見える枠線）
- ホバー背景: アルファ値を20%に設定（ほんのりと色がつく背景）

**ホバー時のテキスト色**（`oklch(from currentColor ...)` 構文）

- 元の色に `currentColor`（現在のテキスト色）を指定することで、CSS変数を使わずに現在色を基準にした計算ができる
- 明度を75%に下げて、ホバー時にアイコンをわずかに暗くする

### 実現している効果

コピーボタンは様々な背景色（配色プレビューのスウォッチ上など）に配置されるため、ベース色は呼び出し元から動的に渡される。`rgb(from ...)` を使うことで、渡された色のRGBチャンネルを再計算なしにそのまま利用しながら半透明化できる。また `oklch(from currentColor ...)` により、ホバー時のテキスト暗化をCSSだけで表現でき、Svelte側のロジックを増やさずに済んでいる。

## 6. ジオパターンセクションのローディングオーバーレイ（`GeoPatternSection.svelte`）

### 使用箇所

幾何学模様パターンを表示するセクションで、パターン生成中にスピナーを表示するオーバーレイ要素の背景色。

```css
background: color-mix(in srgb, var(--color-surface, #f5f5f5) 80%, transparent);
```

### 色の操作

- `color-mix(in srgb, <色A> <割合>, <色B>)` は、2色をsRGB色空間で混合する構文
- `--color-surface`（サーフェス色。デフォルト `#f5f5f5`）を80%、`transparent`（透明）を20%で混合することで、80%の不透明度を持つサーフェス色を生成する

> `color-mix` は相対色構文とは異なる構文ですが、同じく CSS の新しい色操作機能の一つです。

### 実現している効果

`opacity` プロパティで要素全体を半透明にすると子要素（スピナー）も透明になってしまうが、`color-mix` で背景色自体を半透明にすることでスピナーは不透明なまま表示できる。背景越しに模様がうっすら透けて見えながら、スピナーはしっかり視認できるオーバーレイが実現されている。

## 7. 配色プレビューのコピーボタン色（`ThemeColorSchemePreview.svelte`）

### 使用箇所

配色スキームのプレビューコンポーネントで、各スウォッチ（ベース・アソート・アクセントカラーの帯）の上に表示されるコピーボタンの色を、スウォッチの背景色に基づいて動的に決定する箇所。

```svelte
{@const light = isLight(item.color.hex)}

<CopyButton
  color={light
    ? `oklch(from ${item.color.hex} calc(l - .30) c h)`
    : `oklch(from ${item.color.hex} calc(l + .30) c h)`}
/>
```

`isLight()` は輝度（relative luminance に近い加重平均）を計算して明暗を判定する関数。

### 色の操作

- **明るい色の場合（`light === true`）**: OKLCHの明度を `-0.30` 下げて、暗めの色を生成
- **暗い色の場合（`light === false`）**: OKLCHの明度を `+0.30` 上げて、明るめの色を生成
- 彩度・色相はそのまま維持する

### 実現している効果

配色プレビューでは、スウォッチの背景色はユーザーが選んだ任意の色になる。コピーボタンのベース色をスウォッチ色と同系色（同じ色相・彩度）に保ちながら、明度だけ反転方向にずらすことで、背景色がどんな色であってもボタンが視認できるコントラストを自動的に確保している。

## 8. トーンセレクターのセル枠線・選択リング（`ToneSelector.svelte`）

### 使用箇所

PCCSトーンを選択するインタラクティブなUIコンポーネントで、各トーンセルの枠線色と、選択中セルの強調リングの色を計算する箇所。

```typescript
// 通常枠線
function getCellStrokeColor(cell: ToneCell): string {
  return `oklch(from ${getFillColor(cell)} calc(l * .85) c h)`
}

// 選択中リングの枠線
function getSelectedRingStrokeStyle(cell: ToneCell): string {
  const fill = getFillColor(cell)
  return `pointer-events: none; stroke: oklch(from ${fill} calc(l - .10) c calc(h - 10));`
}
```

### 色の操作

**通常枠線**

- セルの塗り色の明度を85%に下げる。彩度・色相は維持。
- ToneDiagram や HueWheel と同様のパターン。

**選択リング**

- 明度を `-0.10` 下げる（ごくわずかに暗くする）
- **色相（`h`）を `-10` 度回転させる**（他の箇所にはないこのコンポーネント固有の操作）
- 彩度はそのまま維持

### 実現している効果

通常枠線は塗り色より少し暗い同系色で、トーン図（ToneDiagram）と同様の効果を得ている。

選択リングについては、色相をわずかに回転させることに重要な意味がある。元の色が白・黄・淡い色など明度の高いものであった場合、単純に明度を下げるだけでは白背景に溶け込んでしまい、選択リングが見えにくくなる。色相を10度ずらすことで、白や明るい黄色からは青緑寄りの色が生成され、白背景上でもリングがはっきり視認できる。彩度はそのまま保持しているため、有彩色のトーンに対しては元の色相から少し回転した隣接色のリングになり、選択状態が色調を壊さずに強調される。

## まとめ

| 使用箇所 | 構文 | 主な操作 |
|---|---|---|
| HueWheel — ハイライト枠線 | `oklch(from ...)` | 明度を×0.85 |
| ToneDiagram — セル枠線 | `oklch(from ...)` | 明度を×0.85 |
| ToneDiagram — ラベル文字色 | `oklch(from ...)` | 明度を±0.60（背景の明暗に応じて） |
| HueSelector — 選択枠線 | `oklch(from ...)` | 明度を×0.75 |
| CopyButton — 枠線・ホバー背景 | `rgb(from ...)` | アルファ値のみ変更（35%/20%） |
| CopyButton — ホバー文字色 | `oklch(from currentColor ...)` | 明度を×0.75 |
| GeoPatternSection — オーバーレイ背景 | `color-mix(in srgb, ...)` | transparentとの混合で半透明化 |
| ThemeColorSchemePreview — コピーボタン色 | `oklch(from ...)` | 明度を±0.30（背景の明暗に応じて） |
| ToneSelector — セル枠線 | `oklch(from ...)` | 明度を×0.85 |
| ToneSelector — 選択リング | `oklch(from ...)` | 明度を−0.10、色相を−10度 |

CSS相対色構文を使うことで、JavaScript側で色計算のロジックを持たなくても、CSSまたはテンプレート内のインライン式だけで「元の色に連動した派生色」を動的に生成できている。特にOKLCH色空間を使うことで、人間の知覚に近い均一な明度スケールで操作できるため、どんな色相・彩度の色を入力しても意図通りの明暗調整が得やすい点が大きな利点となっている。
